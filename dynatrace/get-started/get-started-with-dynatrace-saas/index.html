    开始使用 Dynatrace SaaS

    使用 OneAgent， Dynatrace可以轻松监控整个应用程序栈。OneAgent 执行与监控主机，进程，应用程序，基础架构和真实用户体验相关的所有任务。有关有价值的 Dynatrace 功能的简短介绍，请参阅让您绝对喜爱 Dynatrace 4 特性。

    您可以在 Dynatrace SaaS 服务水平协议中了解我们对监控服务质量和正常运行时间的承诺。

    开始使用Dynatrace SaaS

    您只需完成几个步骤即可启动全栈监控。

    第1步：创建您的Dynatrace帐户

    您需要做的第一件事是创建一个帐户。为此，请访问 Dynatrace 网站并单击“免费试用”按钮。系统将提示您提供电子邮件地址，密码，联系信息以及您所在的地区。如果您在亚洲，您还可以访问这里，申请免费试用。

    如果单击“Deploy Dynatrace”，则可以开始安装。但是，在安装之前，您可以选择在安装 OneAgent 之前查看一些示例监视数据以获取 Dynatrace 的预览。要查看基本的 Dynatrace 仪表板，导航菜单和切片，请单击查看示例数据链接（请参见上图）。然后，您可以单击任何示例仪表板切片（使用示例标签标记）以预览 Dynatrace 的工作方式。

    完成此步骤后，您基本上已在 Dynatrace 集群上创建了监视环境。

    第2步：安装OneAgent和Environment ActiveGate

    如果您的网络允许与 Dynatrace 群集进行 SSL 通信，则可以安装 OneAgent。否则，您需要调整网络设置以允许将监控数据传输到 Dynatrace SaaS 环境。在这种情况下，建议您在安装 OneAgent 之前安装 Environment ActiveGate。环境 ActiveGate 充当 Dynatrace OneAgent 和 Dynatrace 集群之间的代理。即使您的网络允许与 Dynatrace 群集进行 SSL 通信，您仍需要为特定用例安装 ActiveGate，例如，如果您要监视大量主机。

    注意：

    • 如果您无法访问 Web 服务器且无法安装 OneAgent，则可以设置以下任何一项：
      • 综合监测。
      • 无代理监控。
      • RUM 浏览器扩展。
    • 对于 OneAgent 无法监控的应用程序，例如传统的富客户端应用程序，数字服务终端（例如，ATM）或智能物联网应用程序，请使用 Dynatrace OpenKit。

    第 3 步：微调您的环境

    您现在可以通过登录 Dynatrace 监控环境来访问您的主仪表板。所有监测数据都应该在那里。但是，需要一些额外的部署后步骤。

    1. 使用 My web application 定义应用程序环境。
    2. 激活合适的 JavaScript 框架以获取其他监视数据。
    3. 确保自动检测到的进程符合您的期望并满足您的需求。如果不是这种情况，您可以调整默认进程组的组成和/或命名。
    4. 根据您的偏好和需求自定义仪表板。您可以为最重要的应用程序创建单独的仪表板，也可以创建一个仪表板，显示最关键的基础架构组件的运行状况。
    5. 设置所需的通知（notifications)。如果要修改要通知的条件，还可以设置自定义警报规则。
    6. 定义管理区域（management zones)以过滤所有 Dynatrace 视图，以便组织的团队仅查看其负责监视的实体相关的指标。
