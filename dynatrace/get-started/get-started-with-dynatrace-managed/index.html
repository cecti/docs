    开始使用 Dynatrace Managed

    Dynatrace 还提供 Dynatrace 的本地部署模型，称为 Dynatrace Managed。使用 Dynatrace Managed，您可以从我们的 Mission Control 团队获得自动化支持。您可以在 Dynatrace Managed 任务控制和服务水平协议中阅读有关任务控制和服务质量以及双边合同义务的信息。

    开始使用 Dynatrace Managed

    • 第一步：获取 Dynatrace Managed 授权

    要获取 Dynatrace Managed 的许可证，请联系 Dynatrace 销售。您的销售代表将为您提供更多详细信息。Dynatrace 监控的许可基于各种类型的监控单元的消耗，您可以根据监控需求购买和使用这些监控单元。有关详细信息，请参阅如何计算监视消耗。

    达成协议后，您将收到一封电子邮件，其中包含许可证详细信息以及有关如何设置Dynatrace Managed 群集的说明。

    • 第二步：设置 Managed 群集

    要正确设置 Managed 群集，首先应确定最适合您需求的部署模型。随后，您可以安装 Dynatrace Managed 以创建第一个群集节点。您可以随时添加更多节点。

    • 第三步：安装 OneAgent 和 ActiveGate

    要监控环境，请在每个环境中安装 OneAgent。您还可以根据所选的部署模型根据需要安装环境 ActiveGates 和群集 ActiveGates。

    • 第四步：微调您的环境

    您现在可以通过登录 Dynatrace 监控环境来访问您的主仪表板。所有监测数据都应该在那里。但是，需要一些额外的部署后步骤。

    1. 使用 My web application 定义应用程序环境。
    2. 激活合适的 JavaScript 框架以获取其他监视数据。
    3. 确保自动检测到的进程符合您的期望并满足您的需求。如果不是这种情况，您可以调整默认进程组的组成和/或命名。
    4. 根据您的偏好和需求自定义仪表板。您可以为最重要的应用程序创建单独的仪表板，也可以创建一个仪表板，显示最关键的基础架构组件的运行状况。
    5. 设置所需的通知（notifications)。如果要修改要通知的条件，还可以设置自定义警报规则。
    6. 定义管理区域（management zones)以过滤所有 Dynatrace 视图，以便组织的团队仅查看其负责监视的实体相关的指标。
