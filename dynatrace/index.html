    我应该使用哪种扩展方法？

    根据部署的体系结构和要实现的目标，您可以选择以下扩展方法：

    • Dynatrace API

    使用 Dynatrace API，您不仅可以自动执行监控任务，还可以将不同类型的数据导出到第三方报告和分析工具。有关更多信息，请参阅 Dynatrace API。

    • Dynatrace SDK

    • OneAgent SDK

      Dynatrace OneAgent SDK 使您可以手动检测应用程序，以扩展没有代码模块可用的框架和技术的端到端可见性。借助 SDK，您可以完全访问所有分析和监控功能，包括自动基线和基于 AI 的根本原因分析。有关更多信息，请参阅 OneAgent SDK。

    • Dynatrace 插件

      • OneAgent 插入Early Adopter

      使用 OneAgent 插件，您可以将 OneAgent 监控扩展到任何 OneAgent 检测到的暴露接口的进程（例如，通过 HTTP 数据库，应用程序和负载均衡提供的进程）。插件在 OneAgent 中运行，并将新技术监控完全集成到 Dynatrace Smartscape 和问题检测中。您需要一些 Python 专业知识来开发 OneAgent 插件。有关更多信息，请参阅 OneAgent 插件。

      • ActiveGate 插件Early Adopter

      使用 ActiveGate 插件，您可以将 Dynatrace 监视扩展到任何公开接口且不允许安装 OneAgent 的远程技术。例如，PaaS 技术、网络设备或云技术。ActiveGate 插件(又称远程插件)在 ActiveGate 上执行，可以从远程数据源获取度量和拓扑，将新技术监控完全集成到 Dynatrace Smartscape 和问题检测中。您需要一些 Python 专业知识来开发 ActiveGate 插件。有关更多信息，请参见 ActiveGate 插件介绍。

      SDK 插件是一个通用的解决方案，它允许您监视任何公开接口的自定义技术。开发和部署插件。Dynatrace 可以无缝地将您的定制技术监视数据集成到相关的分析视图中，充分利用您最喜欢的所有 Dynatrace 特性和 AI 驱动的根本原因分析。

      • 如果可以在主机上安装 OneAgent，请使用 OneAgent 插件。
      • 如果您需要监控无法选择 OneAgent 安装的远程技术，请使用 ActiveGate 插件。
      • 如果您只想监视公开 MBean 的 Java 应用程序，请使用 JMX 插件。

    • 适用于 Android 和 iOS 的移动 SDK

      适用于 Android 和 iOS 的移动 SDK 允许您报告有关应用中移动用户会话的其他详细信息。通过这些 SDK，您可以创建自定义用户操作，测量 Web 请求，报告错误以及标记特定用户。有关更多信息，请参阅适用于 Android 的 OneAgent SDK 和适用于 iOS 的 OneAgent SDK。

    • Dynatrace OpenKit

      如果你的企业在你的应用程序之外有数字接触点，你可以使用 Dynatrace OpenKit 来测试它们，不管它们是否是传统的富客户端应用程序、智能物联网应用程序，甚至是 Alexa 技术。Dynatrace OpenKit 最适合通过 HTTP 通信的客户机/服务器应用程序(例如，富客户机应用程序、嵌入式设备和终端)。有关更多信息，请参见 Dynatrace OpenKit。

    • JMX/PMI 插件

    JMX 技术（Java Management Extensions）是监视使用 Java 构建的应用程序的理想选择。使用 Dynatrace JMX 监视插件，您可以监视 JVM 中通过 MBean 公开的任何指标。有关更多信息，请参阅如何在 Java 应用程序中监视 JMX 度量标准？。

    • 自定义指标摄取和分析

    自定义度量标准定义为一种新类型的度量标准，它提供用户提供的度量标识符和度量单位。自定义度量标准的语义由您定义，不包含在默认的 OneAgent 安装中。根据自定义指标的定义，可以报告多个受监控组件的指标。要安排自定义度量标准提取和分析，请联系 Dynatrace 销售。
