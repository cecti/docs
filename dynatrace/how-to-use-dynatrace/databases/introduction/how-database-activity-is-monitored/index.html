    如何监视数据库活动

    在监视数据库中，Dynatrace 不考虑方法，而是考虑与数据库服务相关的提交，查询和修改。通过监视此类调用，Dynatrace 能够针对数据库服务的性能问题提供自动根本原因分析。例如，如果您的数据库查询或提交速度变慢，我们会立即通知您。我们甚至会告诉您哪些服务受到问题的影响。

    只要与数据库的交互依赖于受支持的数据库框架（如 JDBC，ADO.NET 或 PDO），就会通过受监视的 Java，.NET，PHP 和 Node.js 进程自动监视数据库调用。在运行应用程序服务器的主机上安装 OneAgent 时，只要对调用进程执行深度监视，Dynatrace 就会确保记录所有数据库语句。一旦监视了对数据库的第一次调用，就会使用新的数据库条目更新“ 数据库”页面。

    在监视数据库活动时，Dynatrace 会显示哪些数据库语句执行得最频繁，哪些语句占用的时间最多。您还可以看到哪些服务执行数据库语句。例如，当一个服务开始发送过多的数据库语句并将数据库服务器推到超出其容量时，Dynatrace 会自动知道。

    当与数据库相关的问题出现时，例如由于响应时间下降，Dynatrace 会告诉您该问题与相关服务上的任何负载增加之间是否存在相关性，而这可能是问题的根本原因。
