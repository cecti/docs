    支持 SQL 绑定变量

    由于网络存储需求较高，对于绑定变量的支持仅适用于 Dynatrace Managed 部署，而不是用与 SaaS 部署。

    绑定变量是参数化 SQL 语句的一种方法，因此语句的 where 子句中有问号或参数，例如:

    • SQL 服务器： select count (*) from report where tenant = @tenant
    • Java JDBC： select count (*) from report where tenant = ?

    绑定变量允许数据库服务器准备语句一次并多次执行，而无需重新解析或分析语句。

    注意：绑定变量不适用于使用文字的语句，例如：

    select count (*) from report where tenant = ‘xxxx’。

    这些语句不能参数化，并且每次执行都会由数据库服务器重新解析或分析。

    注意：这个特性可以捕获很多敏感数据，这就是为什么它只在 Dynatrace Managed 环境中可用。因为它可以产生高的网络流量和存储需求，所以建议谨慎使用它，并且只针对特定的进程组。

    启用 SQL 绑定值捕获：

    1. 在Dynatrace Managed 群集管理控制台导航菜单中，单击“设置”>“服务器端服务监视”>“深度监视”。

    2. 在“ 数据库”下，启用“ 捕获 SQL 绑定值”设置。

    无论您是在站点范围内启用此设置，还是为单个进程组启用此设置，都可以使用 Dynatrace OneAgent 捕捉绑定参数的值。这适用于下列技术:

    • ADO.net
    • JDBC
    • PHP database frameworks
