    用户会话

    用户会话(有时称为“访问”)是在 web 应用程序中有限时间内执行的一组用户操作。一个会话通常包括多个页面加载、第三方内容请求、服务请求和用户操作(例如，单击按钮和下载文件)。每个用户会话至少包含一个用户操作。

    实时用户会话通过颜色与已结束的用户会话区分开来，从而使您可以一目了然地查看活动会话。

    要查看用户会话数据，请从导航菜单中选择用户会话。

    Dynatrace 通常显示每个用户的所有会话，即使这些会话是匿名的(通过浏览器 cookie 标识的未经身份验证的用户)，或者用户标记已被编辑或删除。对于移动应用程序，Dynatrace 根据用户使用的特定移动设备来识别个人用户。对于 web 应用程序，用户标识是通过在每个用户的浏览器中存储一个持久 cookie 来实现的。cookie 允许 Dynatrace 将匿名用户会话分配给已知用户。只要用户至少登录您的应用程序一次，您就可以搜索并标识该用户，即使该用户在匿名、未经身份验证的会话中访问您的应用程序。这对于分析由于身份验证服务问题导致用户无法登录应用程序的时间段特别有用。

    用户会话何时结束

    用户会话以下列方式之一结束：

    • 浏览器不活动30分钟后。
    • 60分钟后（无论浏览器活动与否）。
    • 当用户关闭浏览器时。请注意，当用户只是关闭浏览器选项卡时，用户会话不会结束。
    • 一旦达到每个会话300个用户操作的限制。
    • 通过调用 JavaScript API 函数dtrum.endSession()。例如，当用户注销时，此功能可在您的应用程序中自动结束用户会话。如果在测试期间，您希望立即结束浏览器会话，也可以使用此 API 调用。要使用此功能，请打开浏览器的控制台并输入呼叫dtrum.endSession()。然后，您的用户会话将在会话结束后2-4分钟显示在用户会话视图中。

    用户操作如何分组到用户会话中

    Dynatrace 在会话 cookie 和本地存储中存储一些信息。这允许将用户操作关联到一个用户会话。但是，当(1)关闭浏览器、(2)用户清除浏览器 cookie或(3)用户清理本地存储时，此会话信息将被删除。在这种情况下，下一个发起的用户操作将启动一个新的用户会话。

    例如，假设用户每1-2分钟执行10次用户操作，并且在每次用户操作之后，用户关闭其浏览器或删除他们的浏览器 cookie。在这种情况下，10个用户会话将出现在用户会话视图中，每个用户会话由单个用户操作组成。

    考虑另一个示例:启动了一个用户会话，但几分钟后用户离开并打开浏览器窗口。如果用户在休息之前关闭浏览器，那么无论离开计算机多长时间，当用户返回应用程序时，都会启动一个新的用户会话。

    如何计算用户会话持续时间

    用户会话持续时间是在会话中启动第一个用户操作与在会话中完成最后一个用户操作之间经过的时间。

    注意：超时时间(不包括用户活动)没有被考虑到用户会话持续时间度量中。

    如何区分实时用户会话与已完成的会话

    实施用户会话呈浅紫色，时间线末尾的箭头指示可以将新用户操作添加到会话中。对于完整的用户会话，没有箭头。

    活动过滤器可用于将活动用户会话与已完成的会话分离。
