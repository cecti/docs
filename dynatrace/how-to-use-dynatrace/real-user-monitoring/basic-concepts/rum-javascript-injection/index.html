    RUM JavaScript 注入

    Dynatrace 真实用户监控（RUM）从客户的 Web 浏览器收集指标，并将浏览器数据与从 OneAgent 获得的服务器端信息相关联。Web 浏览器数据由 JavaScript 标记收集，该标记位于应用程序网页的 HTML中。我们将此 JavaScript 代码放置称为“注入”。

    JavaScript 注入如何工作

    您可以通过以下两种方式之一启动 Dynatrace JavaScript 标记注入：

    • 在安装 Java，Apache HTTP Server，IIS，NGINX 和 Node.js 期间，会自动执行 OneAgent JavaScript 标记注入。启动注入所需要做的就是在 OneAgent 安装后重新启动 Web 服务器进程。OneAgent 识别 HTML 内容响应并自动将小 JavaScript 标记注入每个页面的 head 部分。
    • 您也可以手动将 JavaScript 标记注入 HTML 代码而无需安装 OneAgent。我们在设置过程中为您提供 JavaScript 标记。您可以将标记粘贴到要监视的页面中。 您可以通过调整应用程序设置来定义标签注入的自定义位置。例如，您可以告诉 Dynatrace 在 HTML 响应中定义的字符串之前或之后注入标记。对于 Java，您可以在最内层或最外层HttpServletResponse，自定义 servlet 或过滤器内指定注入。

    访问自定义注入设置

    1. 单击导航菜单中的“ 应用程序 ”。
    2. 选择要配置的应用程序。
    3. 单击浏览按钮（...）并选择编辑。
    4. 单击高级设置。
    5. 单击右侧的 JavaScript 注入。

    根据您启用 RUM 的方式，监控数据将在不同的请求类型中发送到Dynatrace：

    • 如果安装了 OneAgent： 我们的 JavaScript 会将数据发送回您的 Web 服务器（Java，Apache，IIS）。OneAgent 对数据执行一些初始监视，然后将其转发到 Dynatrace 服务器。使用 XHR Post 请求执行数据传输。

    • 如果未安装 OneAgent： 数据将直接发送到 Dynatrace，绕过您的基础架构。

    JavaScript 标签注入了哪里

    Dynatrace JavaScript 只适用于有效的 HTML。当监控数据流崩溃时，混乱的页面代码常常是罪魁祸首。因此，要特别注意基本的 HTML 最佳实践(即，没有未关闭或丢失的标签)。

    我们的 JavaScript 标记仅注入 HTML。我们不修改图像，CSS，REST 服务或 XML 文件。OneAgent 总是尝试将 JavaScript 标记作为页面上的第一个脚本注入。这有助于保持应用程序的完整性，但也可以保证更可靠的监控结果（例如，计时）。如果手动注入 JavaScript 标记，请确保它是每个页面上的第一个可执行脚本。
