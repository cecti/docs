    会话重播

    Session Replay 是一款功能强大的工具，可以实现数字体验监控（DEM）战略的现代化。您可以使用它来捕获并可视地重放每个用户的完整数字体验。

    您可以记录与 Web 应用程序的所有客户交互，并在类似电影的体验中重放每次单击和操作。Session Replay 还使您的 QA 团队可以轻松重现生产问题，开发人员可以使用这些问题来弥合代码和用户体验之间的差距。

    会话重播有助于识别应立即修复的错误以及其他问题，例如格式错误的页面和无限旋转。

    会话重播还可用于识别和分析应用程序中的难点区域，并提高其整体可用性。

    会话重播的用途

    深入错误分析

    使用会话重播，您可以进一步深入了解检测到的错误的详细信息：

    • 检测 JavaScript 错误和其他问题
    • 了解导致错误的确切用户操作
    • 了解问题的严重性及其对用户体验的影响
    • 通过重播和查看会话来观察客户影响（在问题不明显的情况下）

    开发人员可以使用 Session Replay 作为查看，分析，重现和修复错误的方法。

    出于错误钻取的目的，没有必要记录所有会话。您可以使用成本和流量控制只记录会话的一个子集。如果要分析的错误不是太零散，即使只记录了20%的会话，也可以检测到。

    默认的数据保留时间框架为35天，应用于这些会话。

    投诉解决方案

    由于会话重放是演示用户实际操作的最佳方式，因此它提供了解决客户投诉的方法。使用会话重播：

    • 查看客户在您的应用程序上的确切行程
    • 确定用户面临的确切问题
    • 提供正确的说明

    要解决客户投诉，最好记录所有会话。但是，为节省存储空间，建议设置较短的保留时间。

    可用性分析

    会话重播还可用于检测和分析以下问题：

    • UX设计不够直观
    • 流程过于复杂，用户往往会在中途离开您的应用程序
    • 应用程序很慢，用户反复点击以转到下一个屏幕
    • 该应用程序无法在所有浏览器上按预期工作
    • 移动网络应用程序提示用户更改手机的方向，但用户不理解提示

    默认数据保留时间适用于这些会话。

    技术限制

    会话重播与基于页面的应用程序、单页面应用程序和使用 IFrame 的应用程序兼容。然而，也有一些限制。
