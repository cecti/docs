    检测 IP 地址，地理位置和用户代理

    Dynatrace 可自动检测 IP 地址，地理位置以及浏览器，设备和操作系统。

    IP 地址

    IP 地址从 web 请求的 HTTP 头和 Dynatrace 实际用户监视信号中自动标识。然而，当使用负载均衡、CDNs 或代理时，HTTP 请求的远程 IP 可能与来自最终用户设备的原始 IP 地址不同。对于这种情况，Dynatrace 监视 HTTP 头文件，这些头文件最常用来标识通过 HTTP 代理、CDN 或负载平衡器连接到 web 服务器的客户机的初始 IP 地址。要查看默认情况下 Dynatrace 监视哪些头文件，请转到全局设置> Web & mobile monitoring -> IP确定。这些头按特定的顺序处理。但是 Dynatrace 允许您更改处理顺序以及添加您自己的头文件。

    请记住，默认情况下，Dynatrace 会屏蔽用户设备IP地址的最后部分。

    地理位置

    Dynatrace 真实用户监控尝试为每个用户会话分配地理位置（城市，地区，国家/地区），以便按位置对用户会话和用户操作进行分组，并在世界地图上显示它们。

    对于Web应用程序，Dynatrace 使用 MaxMind Geo2 数据库将IP地址映射和解析到地理位置。

    • 当 IP 地址无法正确解析时，如何请求更新 MaxMind 数据库？

    您可以通过https://support.maxmind.com/geoip-data-correction-request/提交表单来请求MaxMind更新其数据库。

    您可以使用https://www.maxmind.com/en/geoip-demo上的表单随时通过MaxMind数据库检查您的IP地址。

    对于移动应用程序，如果应用程序具有使用此信息的权限，则 Dynatrace 将使用设备中的坐标（GPS或 wifi），然后 Dynatrace 将计算最接近报告的 GPS 位置的城市。如果移动应用程序无权访问移动操作系统上的地理位置信息，则它使用 IP 地址再次与 MaxMind Geo2 数据库组合确定地理位置。

    默认情况下，Dynatrace 还会屏蔽 GPS 坐标。

    浏览器/设备/操作系统

    对于 web 应用程序，根据浏览器发送的用户代理字符串将真实用户会话与合成用户会话和机器人会话区分开来。此字符串还用于标识操作系统和设备类型，如桌面、平板电脑或移动设备。对于浏览器分类，Dynatrace 使用 www.udger.com 用户代理数据库。基于 IP 地址检测 isp。Dynatrace 使用 Gbp.potoroo.net 和 www.cidr-report.org 提供的信息来识别给定 IP 地址的 ISP。

    有关 ISP 检测的更多信息

    • Dynatrace 使用 Gbp.potoroo.net，它提供了两个表

    • http://bgp.potaroo.net/as2.0/bgptable.txt 用于 IPv4 地址

    • http://bgp.potaroo.net/v6/as2.0/bgptable.txt 用于 IPv6 地址

    这些表包括 CIDR 块（IP 范围）及其相应的 ASN 查找。每行的最后一组数字是给定 IP 地址范围的 ASN。

    Dynatrace 可接着使用提供的另一个表 cidr-report.org，其中包含了互联网上所有的 ASN 与相应的描述一起。

    这些是平面文本文件（实际上格式化为 HTML），由 Internet 上的提供商定期更新。

    总结一下，Dynatrace 为 Gbp.potoroo.net 中的特定 IP 地址编制 ASN，然后在 cidr-report.org 表中查找 ASN 以查找 ISP。

    Synthetic仅用于 Dynatrace 综合监控，例如浏览器监视器或经典产品（Gomez，Keynote 或 AppMon合成）。

    关于移动应用程序，设备名称由 Android 版 Google Play 商店提供，而对于 iOS，Dynatrace 维护一个交叉引用列表，将 Apple 设备 ID 映射到 Apple 设备名称。操作系统-Android 或 iOS-由 Dynatrace OneAgent for mobile 提供。对于 Web 应用程序，通过在 Gbp.potoroo.net 和 www.cidr-report.org 中查找用户的设备 IP 地址来检测 ISP。
