    应用

    Dynatrace 中的应用程序是客户应用程序(web站点、移动应用程序等)映射到其上的逻辑结构，用于监控来自真实用户的流量。因此，这意味着此类客户应用程序具有最终用户界面。典型的终端用户界面包括基于浏览器的 web 应用程序或 web 站点界面，以及基于 iOS 或 android 的运行在智能手机或平板电脑上的移动应用程序界面。

    最终用户界面决定了在 Dynatrace 中创建的应用程序的类型。Dynatrace 将如何获取监视数据，以及将收集哪些数据，这对于每种应用程序类型都是不同的。例如，对于在浏览器中运行的 web 应用程序，需要在应用程序的 HTML 页面中添加或注入 JavaScript 代码(Dynatrace RUM JavaScript)。有关每种应用程序类型，请参阅下面的详细信息。

    对于除web应用程序之外的所有应用程序类型，当客户创建应用程序并开始设置 Dynatrace 监视时，将生成 Dynatrace 应用程序的定义。这些应用程序类型没有以自动化的方式进行监视。对于 web 应用程序，这个定义是自动生成的。更具体地说，如果您的 web 应用程序运行在可以安装 Dynatrace OneAgent 的系统上，那么 Dynatrace 将自动将 RUM JavaScript 代码注入HTML页面。因此，每个监视环境都会自动获得一个名为“My web application”的默认应用程序。默认情况下，所有的 RUM 数据都分配给这个应用程序。

    Dynatrace 支持的应用程序类型

    Dynatrace 支持不同的应用程序类型: web、移动、AMP以及富客户端应用程序，这些应用程序可以运行在汽车上，也可以运行与用户交互的物联网应用程序。每种类型都与不同的监视功能以及 Dynatrace 中的不同UI相关联。然而，所有类型都被用户会话和用户操作等常见概念所渗透。

    所有 HTML 页面(如静态 web 页面或在浏览器上运行的单页面应用程序)都被视为 web 应用程序。AMP应用程序也是 web 应用程序，但它们遵循 AMP(谷歌加速移动页面)规范，该规范不允许完全的深度监视，因此无法获得相同的数据。移动应用程序可以是 iOS 或 Android 上的原生移动应用程序，也可以是通过浏览器访问的混合应用程序。其余的应用程序关注您环境中的所有数字接触点，从传统的富客户机应用程序到智能物联网应用程序，甚至Alexa技能。这些应用程序是通过 Dynatrace OpenKit 支持的。

    在下文中，为每种应用类型提供了更多信息。

    • Web 应用程序
    用户界面 启用JavaScript的浏览器（移动或桌面）
    监测方法 Dynatrace RUM JavaScript
    注射类型 注入 Dynatrace RUM JavaScript 可以由 OneAgent 自动完成，也可以手动进行无代理监控或使用 Dynatrace 浏览器扩展。
    如何开始 通过“My web application”占位符定义您的应用程序
    • AMP 应用程序
    用户界面 启用 JavaScript 的浏览器（移动或桌面） - 页面遵循 AMP 规范
    监测方法 Dynatrace RUM JavaScript
    注射类型 通过将特殊 AMP 设置传递到 AMP 页面，客户手动完成注入。
    如何开始 设置AMP监控
    • 移动应用
    用户界面 iOS 或 Android 上的原生移动应用
    监测方法 适用于 iOS或 Android 的 OneAgent; 对于混合应用程序，浏览器部分由 Dynatrace RUM JavaScript 监视
    注射类型 不需要注入 - 生命周期事件，用户操作和 Web 请求都是开箱即用的。
    如何开始 开始使用Android监控 开始使用iOS监控
    • OpenKit 授权的应用程序（自定义应用程序）
    用户界面 客户希望通过真实用户监控从富客户端应用程序监控的任何数字接触点，通过汽车中运行的应用程序，通过用户交互进行物联网应用程序。用户界面甚至可以是亚马逊 Alexa 等硬件。
    监测方法 Dynatrace RUM JavaScript
    注射类型 注入由客户自己使用 Dynatrace OpenKit 技术完成。
    如何开始 使用 Dynatrace OpenKit 检测应用程序
