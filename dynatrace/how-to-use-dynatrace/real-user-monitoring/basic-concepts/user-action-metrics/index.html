    用户操作指标

    对于 Web 应用程序，Dynatrace 会计算以下用户操作指标。

    里程碑指标

    里程碑指标表示在用户操作期间特定时间点发生的事件。但是，它们是以毫秒为单位度量的，因为它们是相对于用户操作开始时间计算的。换句话说，用户操作开始时间作为一个参考点，从这个参考点度量里程碑指标的时间。例如，导航启动的47毫秒值表示该事件发生在操作启动时间之后47毫秒。

    度规 描述 适用于
    Navigation start 在相同的浏览上下文中，unload 提示符后的那一刻，在前一个文档上终止了(如果没有前一个文档，该值与performancetimes.fetchstart相同)。 加载动作
    Request start 用户代理从服务器，相关应用程序缓存或本地资源发送获取实际文档的请求的那一刻（如果在请求开始后传输层失败并且重新打开连接，则此属性将设置为对应于新请求的时间）。 加载动作
    Time to first byte 从服务器，相关应用程序缓存或本地资源接收响应的第一个字节的那一刻。 加载动作XHR动作
    HTML downloaded 接收当前页面的最后一个字节的时刻，或者在传输连接关闭之前（以先发生者为准）。 加载动作
    DOM interactive 在页面的准备状态设置为“交互”之前（即，在页面准备好接收进一步的用户输入之前），解析器在主文档上完成其工作的那一刻。 加载动作
    speed index 表示在视觉上填充页面内容的速度的分数。低速索引意味着页面的大多数部分都可以快速呈现。如何计算速度指数？ 加载动作
    Visually complete 浏览器可见区域中的所有内容完全呈现的那一刻。如何计算视觉完成？ 加载动作XHR动作
    DOM contect loaded 用户代理DOMContentLoaded在文档处触发事件的那一刻。 加载动作
    DOM complete 用户代理将当前文档准备就绪设置为“完成”的那一刻。 加载动作
    Load event start 为当前文档发送加载事件的那一刻。如果尚未发送此事件，则返回0。 加载动作
    Load event end 当前页面的加载事件完成的那一刻。如果此事件尚未发送或尚未完成，则返回0。 加载动作
    User action duration 页面加载完成的那一刻。如何计算用户行动持续时间？ 加载操作XHR操作自定义操作

    注意：

    Speed Index 和 Visually Complete 仅适用于支持 mutationobservers 的浏览器。这包括以下浏览器：

    • Microsoft Internet Explorer 11
    • Microsoft Edge 15或更高版本
    • Firefox 57或更高版本
    • 谷歌浏览器61或更高版本

    阶段指标

    与里程碑指标相反，阶段指标意味着持续时间。

    度规 描述 适用于
    Blocking 从浏览器意识到需要下载资源到实际开始下载资源之间的时间。这包括等待空闲 TCP 套接字所花费的时间、生成磁盘缓存项所需的时间和代理协商所花费的时间。 Load actions
    Application cache 用于检查任何相关应用程序缓存的时间。它包括与服务器建立连接之前的时间。 Load actions
    DNS lookup 用于查找当前文档的域名的时间。 Load actions
    TCP connect 用于建立到服务器(包括 SSL)的连接的时间。 Load actions
    Secure connect 建立到服务器的安全连接所花费的时间。这包括 SSL 握手和 SOCKS。 Load actions
    Request 从服务器请求当前文档所花费的时间，直到接收到第一个字节为止。 Load actions
    Response 用于接收响应的时间。 Load actions
    Processing 从 DOM 加载到加载事件启动之间的时间。 Load actions
    OnLoad 用于处理加载事件的时间。 Load actions
    OnDOMContentLoaded 用于执行onDomContentLoaded处理程序的时间。 Load actions
    Response end 从接收到服务器响应的第一个字节到接收到响应当前文档的最后一个字节或响应完成(以先发生的情况为准)所花费的时间。 XHR actions Load actions
    Redirect time 用于后续 HTTP 请求的时间。 Load actions

    关键绩效指标

    从上面列出的指标中，Dynatrace 指定了一些作为关键。关键性能指标使您能够为每个应用程序选择理想的用户体验指标。这些关键指标为您提供了有价值的性能洞察，从而促进您的数字业务的成功。主要性能指标包括:

    • User action duration
    • Visually complete
    • Speed index
    • DOM interactive
    • Load event end
    • Load event start
    • HTML downloaded
    • Time to first byte

    Dynatrace 允许您为每种应用条件选择正确的性能指标。
