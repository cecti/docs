    使用主机组组织环境

    注意：该特性目前是早期采用的版本。

    可以在以下步骤中将主机分配到主机组：

    • 在安装 OneAgent 时，使用HOST_GROUP参数将主机分配给一个组。/bin/sh Dynatrace-OneAgent-Linux-1.137.65.sh HOST_GROUP=MyHostGroup在 Windows 上，您还可以在安装 OneAgent 时输入组名。
    • 在安装 OneAgent 之后使用 oneagentutil 命令行界面

    主机组字符串可能只包含字母数字字符、连字符、下划线和句号。它不能从dt.开始。最大长度为100个字符。

    主机组静态分配给主机。每个主机最多只能属于一个主机组，只能通过重新安装 OneAgent 或使用“oneagentutil”命令行界面来更改主机组。例如，在“监控概述”页面上显示主机组。单机“主机组”链接编辑主机组中所有主机的设置。

    主机组如何影响您的监控环境

    主机组是主机的集合。每个组都可以在主机组级别上配置。这使得更改大量主机的设置变得很容易。您可以根据每个主机组定义警报阈值和一个代理更新设置。在下面的示例中，主机组接受全局配置的异常阈值，而不覆盖它们。
