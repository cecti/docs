    我在哪里可以查看主机性能指标

    单击主仪表板上的“ 主机”磁贴以查看“ 主机”页面 - 环境中检测到的所有主机的列表。

    “ 主机”页面列出了环境中安装了 OneAgent 的所有计算机（物理和虚拟）。单击主机以转到该主机的专用主机页面，您可以在其中查看主机的所有可用指标。

    下图显示了磁贴查看模式下的“ 主机”页面。您可以使用右上角的列表选择要在所有主机上分析的特定度量标准（CPU，内存，磁盘延迟或网络流量）。需要注意的主机显示为红色并显示在磁贴顶部。例如，此图像显示的主机已超出可用磁盘空间阈值。

    下图显示了处于表查看模式的主机页面。单击任何列标题，根据特定的条件对列表进行排序。例如，列表是根据映像中的 CPU 使用情况排序的。

    个人主页上包含那些内容

    每个主机页面都详细说明了所选主机所依赖的硬件资源的运行状况。单击四个运行状况统计信息（CPU，内存，磁盘或 NIC）中的一个，以查看有助于每个度量的度量标准的详细信息。

    下图显示了“ 主机”页面上的网络接口流量。

    问题磁贴可让您查看与所选主机相关的活动和已关闭问题。Dynatrace 还会将重复出现的问题模式报告为此磁贴中的常见问题，但只有在周期性模式的严重性增加时才会发出警报。

    单击列表中的问题以查看问题的根本原因并了解其对您的服务的影响。

    除了性能指标和图表，您还可以在主机页面可用性磁贴上跟踪主机可用性。这表示主机在线并响应请求的时间百分比。Dynatrace 检测并显示操作系统关闭（包括重新启动）和主机脱机的时间段（例如，如果主机意外关闭）。Unmonitored 表示监视关闭的时间段。

    深入挖掘性能因素

    在每个主机页面上，您将找到一个或多个按钮，指示您显示有助于所选主机运行状况统计信息（贡献进程，贡献磁盘或贡献网络接口）的特定组件的详细信息的页面。这张图片显示了内存使用情况链接，链接到有助于内存使用健康统计的组件进程列表。

    这幅图像显示了对这个健康统计数据做出贡献的组件进程。

    单击特定组件流程，查看其属性和 Dynatrace 为该组件捕获的性能指标。process 页面上的进程磁贴和特定的进程概述页面都会提醒您任何需要重新启动的进程。

    进程磁贴还显示主机 Docker 容器的信息，并允许您深入到 Docker 详细信息页面，以查看每个映像中容器的 CPU，内存，流量和限制详细信息。
