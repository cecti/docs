    主机健康测量

    各个主机页显示每个主机的问题历史、事件历史和相关流程。要评估健康状况，为每个主机捕获以下性能指标，并在每个主机概览页面上显示:

    • CPU
    • 内存
    • 磁盘（存储健康）
    • 网卡（网络健康）

    CPU健康

    CPU使用率是用于计算 CPU 运行状况的主要度量。这是 CPU 忙于处理数据的时间百分比（即，它不是空闲时）。此百分比是针对所有可用 CPU 核心计算的，并缩放到0–100%的范围。

    系统的总 CPU 使用量和特定进程组的 CPU 使用量使用相同的计算方法。这意味着由4核系统上的单线程进程组成的进程组将达到最大 CPU 使用率25% (4 x 25% = 100%)。

    高 CPU 使用率测量会导致 CPU 饱和“资源事件”并产生问题。

    CPU 使用率测量每10秒捕获一次。每个10秒间隔的平均 CPU 消耗用于计算总的 CPU 使用量。因为 Dynatrace 在10秒间隔内平均 CPU 消耗，所以在10秒周期内发生的 CPU 消耗的瞬时波动可能会被消除，但是在每个10秒周期内的平均 CPU 消耗是准确的。

    • 虚拟主机的附加性能测量

    虚拟主机显示与 CPU 性能相关的其他测量，这些值对整个虚拟机的运行状况很重要。

    • CPU 就绪时间

      虚拟机准备好，但未计划在物理CPU上运行的时间百分比。

      注意：CPU 准备时间应保持在10%以下。CPU 就绪时间度量超过10%表明您的虚拟机正在争夺可用资源，虚拟机无法执行它的所有任务。这种争用可能导致应用程序性能下降。有关更多信息，请参见虚拟机迁移如何影响性能?

    • CPU AWS

      OneAgent 从 AWS 本身的角度报告的 CPU 消耗，而不是来自主机内部的 CPU 消耗。

    • 物理 CPU 主动使用虚拟 CPU 的数量占总可用 CPU 的百分比。这是 CPU 使用情况的主机视图，而不是客户操作系统视图。它是虚拟机上所有可用虚拟 CPU 的平均 CPU 利用率。例如，如果一台拥有一个虚拟 CPU 的虚拟机运行在一台拥有四个物理 CPU 的主机上，并且 CPU 使用率为100%，那么您就知道虚拟机正在使用一个物理 CPU 的100%可用资源。

    • AIX 主机的其他性能测量

    AIX主机显示与CPU性能相关的其他度量。这些值对整个虚拟机运行状况很重要。

    • 权限配置

      LPAR 定义中的可配置设置，您可以在其中指定将分配给此 LPAR 的物理 CPU 核心的默认最小数量。此指标反映的是设置值，而不是实际消耗。消耗可能因物理 CPU 的可用性而异。

    • 权限使用

      已使用的配置权限的百分比。考虑到 LPAR 接收的 CPU 可能比您在授权中配置的 CPU 多，这个值可以达到100%以上。

    • 消耗

      消耗的物理 CPU 核数。

    • 线程运行

      在采样期间可运行内核线程的平均数量。这是等待运行或已经运行的线程数。

      一个稳定的工作负载通常运行少于五个线程。运行的线程数的快速增加可能表明存在应用程序问题。争夺 CPU 资源的线程以循环方式调度。如果每个线程执行一个完整或部分时间段，那么可运行线程的数量可能超过100。

    • 线程被阻塞

      在采样周期内放入等待队列的平均内核线程数。当内核线程被安排执行并等待其中一个进程页面被请求时，它们被添加到等待队列中。

    • I/O 消息等待

      来自原始设备的等待 I / O 消息的平均线程数。原始设备是直接连接到系统的设备。

    • I/O direct, buffered

      每秒等待文件系统直接 I/O 事件发生的线程数和等待缓冲 I/O 的进程数。这个度量的值可能与实际值不同。它表示六个元素中两个元素之和，这两个元素等于实际值。

    • 虚拟 CPU

      分配给逻辑分区的物理 CPU 核心数。

    • 逻辑 CPU

      通过将同步多线程（SMT）技术应用于每个虚拟处理器得到的处理器数。例如，2个虚拟处理器，每个处理器有4个 SMT 线程，提供8个逻辑 CPU。

    • 同步多线程（SMT）

      执行的独立线程数（以更好地利用处理器资源）。

    内存健康

    主机页面包括两个与主机相关的内存相关指标，内存使用 和页面错误。测量和其他因素都用于关联和计算主机高内存事件。

    • 内存使用

    进程使用的总 RAM 百分比。系统缓存和缓冲区使用的 RAM 不包含在此度量标准中。Dynatrace 计算内存使用情况为： memory_used = total_memory_size - (free_memory + active_memory + inactive_memory + reclaimamble_memory)

    • 页面故障

    每秒的主要页面故障的次数。主要页面错误涉及从磁盘加载页面，从而为中断程序的执行添加磁盘延迟。

    • 虚拟主机的附加性能测量

    虚拟主机显示与虚拟机内存使用相关的其他度量。这些度量以及其他度量用于检测内存饱和事件。

    • 内存压缩

      内存压缩或解压缩的速率。虚拟机管理平台使用内存压缩来减少内存使用。内存压缩可节省内存，但需要额外的 CPU 周期。先前已压缩的内容必须先解压缩，然后才能被虚拟机使用。

    • 内存交换

      内存从磁盘交换到活动内存的速率，反之亦然，从活动内存交换到磁盘。

    磁盘健康

    磁盘健康包括：

    • 吞吐量

    每秒读取和写入磁盘的总字节数。

    • IOPS

    每秒 I/O（输入/输出）操作。操作是在对相邻磁盘扇区进行寻址操作合并后计算的。

    • 磁盘延迟

    从 I/O 请求提交到 I/O 请求完成的时间。磁盘读写操作的平均延迟（以毫秒为单位）。此度量标准用于检测主机慢速磁盘事件。

    • 磁盘空间的使用

    所使用的磁盘空间量。

    • 空闲时间

    磁盘空闲的时间量。

    网卡健康

    网卡健康包括：

    • 流量 在间隔期间传输数据的平均速率。

    • 数据包 间隔期间主机网络接口上接收和发送的数据包数。

    • 质量 评估丢弃的数据包和错误的数量。

    • 连接

    与被拒绝或超时的 TCP 连接相比，正确建立的 TCP 连接的百分比。

    注意：连接性度量可用于指示主机上是否有网络流量。但是请注意，0%的连通性并不一定表示主机有问题。假设没有出现 TCP 错误，这可能只是意味着在选定的时间段内没有用户尝试连接到主机进程。
