    OneAgent 和 ActiveGate发行说明版本 1.171

    OneAgent

    Java

    • 支持缩减的 Apache Axis2 客户端（例如，与 Pega 一起打包的客户端）
    • 支持在 Spring 集成中跟踪异步代码执行

    .NET

    • 通用可用性（GA）版本中提供了对 Microsoft Azure Service Bus for .NET 的支持，为 Azure ServiceBus 队列和主题提供端到端跟踪。
    • 现在，GA 版本中提供对 .NET Remoting 的支持，为旧版 .NET Remoting 应用程序提供端到端跟踪。
    • 现在，GA 版本中提供了对 IBM MQ Client 的支持，使用 .NET 客户端库为 IBM MQ 提供端到端跟踪。
    • OneAgent SDK for .NET（版本 1.5.0）现在支持跟踪传入的 Web 请求。
    • .NET Instrumentation Cache 可以改善启动延迟，现在是通用可用（GA）。

    Node.js

    • 支持 Node.js 12（LTS 版）

    Go

    • 支持 Go 版本 1.11.10 和 1.12.5

    PHP

    • 支持 PHP NG（PHP下一代）监控现在可作为 Early Adopter 版本提供。

    一般改进

    • OneAgent 版本 1.171 不需要特殊许可证密钥即可安装到云基础架构监控模式。在此之前，一年前获得 Dynatrace 许可证的客户必须先更新许可证才能使用云基础架构监控模式。此要求已被删除; 现在可以在全栈或云基础架构模式下安装 OneAgent。
    • 现在，OneAgent 支持并传播 W3C 跟踪上下文头，有关详细信息，请参阅使用 W3C 跟踪上下文进行分布式跟踪，以提高端到端可见性（预览）
    • 改进了 Citrix 进程的分组
    • 改进了 CRI-O 版本 1.12.6+ 中钩子目录的检测
    • 在 Windows 上报告 Kubernetes 集群主机的云平台类型和版本
    • 检索 containerD 元数据时，大大提高了性能，在监视 containerD 主机期间减少了 OneAgent CPU 消耗。
    • 增加了一种用于发现延迟加载 .NET 应用程序的改进启发式算法。
    • Windows Diego Cell 发现启发式算法已得到改进，以匹配最近的 Garden for Windows 版本中的更改。

    OneAgent 支持变化

    Dynatrace OneAgent 对以下平台和组件的支持将在未来 6 个月内停止使用 - PHP 5.6 支持已经延长，并将在 2019 年 11 月达到使用寿命 - PCF PAS版本 2.2.x（该版本的关键支持于 2019 年 4 月结束。有关详细信息，请参阅 Pivotal 支持矩阵）。OneAgent 版本 1.171 是支持PCF版本 2.2.x 的最后一个版本。 - PCF PAS 版本 2.3.x（此版本的关键支持将于 2019 年 6 月结束。有关详细信息，请参阅 Pivotal 支持矩阵）。OneAgent 版本 1.175 是支持 PCF 版本 2.3.x 的最后一个版本。 - WebSphere Application Server V8.0（IBM 于 2018 年 4 月发布的 EOS）OneAgent 1.183 将是支持 WebSphere v8.0 的最后一个版本。 - Java 6 WebSphere Application Server V8.5（IBM 于 2018 年 4 月发布的 EOS）OneAgent 1.183 将是支持带有 Java 6 的 WebSphere V8.5 的最后一个版本（之后，WebSphere 8.5 只能与 Java 7 一起使用）

    云平台支持

    AWS

    我们对 AWS CloudWatch 监控组件进行了多项改进，以避免 API 限制。

    Cloud Foundry

    Dynatrace 现在还支持 Pivotal Cloud Foundry PAS 2.6 版和 OneAgent BOSH 版本 1.2.1。有关详细信息，请参阅用于全栈 PCF PAS 监视的 Dynatrace 支持模型。 我们在 Cloud Foundry 概述页面上改进了 Diego 细胞的呈现方式。您现在可以更轻松地跟踪 Diego 单元的 CPU，内存和磁盘利用率。

    Kubernetes

    Dynatrace 现在支持带有 CRI-O 容器运行时的 Red Hat OpenShift 4.1。 我们在 Linux 上引入了 Kubernetes 的容器监控规则作为早期采用者版本。容器监视规则允许您利用 Kubernetes 属性来控制 OneAgent 自动注入 Kubernetes 容器。这有助于在基于组织结构的大规模环境中系统地推出全栈监控。

    ActiveGate

    ActiveGate 支持变化

    以下是 ActiveGate 的解除支持计划

    • OneAgent 版本 1.177 将是仅支持 32 位主机 ID 的最后一个版本。到那时，必须将早于 1.144 版本的所有 ActiveGates 升级到较新版本才能正确支持 OneAgent 64 位 ID。如果不这样做，将导致 OneAgent 无法通过这些早期版本的 ActiveGate 与 Dynatrace 群集进行通信。除此之外，这种变化将是完全透明的。

    一般

    • 一个插件循环中使用的线程数量增加。（APM-174982）
    • 修复了构建插件时递归文件复制的问题。（APM-175675）
    • 请求库版本更新（APM-179383）
    • 修复了 Internet Explorer 卸载帧时发生的 Visually Complete 模块中的问题（APM-173974）
    • 修复了如果用户输入已用于创建操作，则创建源操作的问题（APM-177232）
    • [PHPAgent]缺少 mysql_unbuffered_query 节点的 QueryAttachment（APM-176248）

    ActiveGate

    • 修复：在安装了 Synthetic（EAP）的 ActiveGate 更新期间，代理流量被禁用。（APM-175642）
    • 私有收集器归档文件不包含来自 supportarchive 目录的日志（APM-175717）
    • 由于代理问题导致 RPM 安装失败（APM-177967）
    • Windows Synthetic 安装程序 - 未安装 version.properties 文件（APM-178089）
    • 没有针对大型 VMware 环境报告 VMware 指标。（APM-178671）
    • 修复了已安装 RPM 的 AG 升级。（APM-178822）
    • 云监控时间序列中的差距（APM-179150）
    • VMWare 正在丢弃稀疏指标的所有度量（APM-179295）

    OneAgent安装程序

    • Linux 安装程序现在将正确禁用 systemd ABRT 服务（APM-177311）
    • Linux 安装程序未正确验证用户创建是否成功（APM-177574）
    • 现在，当使用非特权模式时，Linux 安装程序将正确检测内核是否不支持文件功能（APM-174347）
    • 修复了 Windows 安装程序中某些引用路径的处理 --unpack-msi 参数（APM-175923）
    • 当环境功能可用且指定了 DISABLE_ROOT_FALLBACK = 1 时，在非特权模式下正在禁用自动更新（APM-177832）

    • 如果远程对象驻留在 LogicalCallContext 中，则.NET Remoting 传感器会导致问题。（ONE-22986）

    • 启用聚合的 ADO.NET 传感器未报告正确返回的行数。（ONE-23923）
    • 修复了 ASP.NET 1.1 应用程序可能发生的代理崩溃 （ONE-24796）
    • Oracle 数据库名称错误，因为连接字符串解析无法正确处理 SID 密钥。（ONE-25339）

    Node.js

    • 在请求重定向的情况下，代理将头部属性转换为小写（ONE-25337）

    Go

    • 修复指标问题，导致 Go 进程在关机期间的罕见情况下崩溃（ONE-24547）

    JavaScript

    • 改进了 ExtJS 模块，以防止客户代码假设 ExtJS 可用并使用不存在的功能。（ONE-24969）
    • 修复了如果 mootools 的配置对象为 null / undefined 时发生的 mootools 模块的问题。（ONE-24669）
    • 修复了使用 Object.freeze 冻结 XHR 配置时出现的问题。（ONE-24697）
    • 修复了视觉完整模块中有关失败图像计算的性能问题。（ONE-25223）

    Apache

    • Apache 使用 mod_headers 和快速内部重定向复制响应头值（ONE-25869）
    • 快速内部重定向（ONE-25870） 后 Apache 代理的子路径链接不正确

    OS模块

    • 修复了关于缺乏对 PHP 进程进行深度监视的误报警告，这些进程与不同类型的静态链接进程组合在一起。（APM-172664）
    • 现在可以在 RedHat 发行版上正确读取进程已删除二进制文件的名称，其中删除时间戳添加到 /proc//exe 符号链接。（APM-172002）
    • 修复了由于二进制静态链接到具有非常大的 /proc/pid/maps 的进程而导致注入可能性不足的误报信息（APM-174460）
    • 已调整 Kubernetes 版本发现以匹配 hyperkube API 中的更改。（APM-178581）
