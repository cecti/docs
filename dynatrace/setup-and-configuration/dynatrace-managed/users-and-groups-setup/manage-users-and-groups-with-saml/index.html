    在 Dynatrace Managed 中使用 SMAL 管理用户和组

    Dynatrace Managed 支持与SAML 2.0集成，作为SSO IdP（单点登录身份提供程序），用于管理用户和组。SAML 可以使用HTTP POST（首选）或HTTP Redirect绑定。当两者都存在时，HTTP POST使用。

    设置SAML 2.0集成

    1. 打开 Dynatrace Managed 群集管理控制台并导航到用户身份验证 > 单点登录设置。
    2. 从下拉列表中，选择 SAML 2.0。
    3. 单击“ 下载 SP 元数据”按钮以下载 SAML 服务提供商元数据文件。
    4. 使用 SP 元数据文件将 Dynatrace Managed 配置为在身份提供程序服务器（IdP）上充当服务提供程序。有关详细信息，请参阅您的 IdP 文档。
    5. 从IdP服务器下载完整的配置元文件。有关详细信息，请参阅 IdP 文档。
    6. 返回 Dynatrace Managed，单击 Select file 按钮并将您的IdP配置元文件上传到 Dynatrace Managed。
    7. 在相应的文本字段中，根据SAML 2.0响应属性键入名字，姓氏和电子邮件的用户属性。

    组分配配置

    必须将每个 Dynatrace Managed 用户分配给至少一个用户组，并且至少具有一个关联的监视环境。没有这样的映射，用户无法登录到 Dynatrace Managed。用户将收到一条错误消息，指出未找到任何环境。

    您可以通过以下两种方式之一管理用户组分配：

    • 手动，从 Dynatrace Managed 中。对于手动用户组分配，请将基于SAML 2.0响应属性的用户分配到组关闭到关闭位置。使用此方法，Dynatrace Managed 将忽略在 IdP 的身份验证响应中发送的组列表。

    • 自动 对于自动用户组分配，将“ 根据SAML 2.0响应属性分配用户组”设置为“ 开”位置，并在“ 用户组属性”字段中键入组名称。使用此方法，在Idat的身份验证响应中发送的组列表将覆盖在 Dynatrace Managed 中进行的任何分配。

    你使用 ADFS 吗？

    要将 Dynatrace Managed 与 ADFS 身份提供程序集成，请执行以下配置：

    ### 在 ADFS 端配置

    1. 使用Dynatrace SP元数据配置添加新的中继方信任。

      然后将安全散列算法设置为 SHA-1。

    2. 向已添加的中继方信任添加“请求颁发策略”。

    3. 定义所需的规则：

      • 将 LDAP 属性作为声明发送的规则
      • 将 LDAP 属性转换为名称 ID 的规则（创建适合您需求的规则）
      • 将 LDAP login属性转换为的规则示例Name ID
      • 将 LDAP email属性转换为的规则示例Name ID

      ### 在 Dynatrace Managed 端配置

      在 Managed Single 登录设置页面上适当设置 User 组属性（用户身份验证 > 单点登录设置）。
