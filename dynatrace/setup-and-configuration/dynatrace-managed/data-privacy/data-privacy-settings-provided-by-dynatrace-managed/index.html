    Dynatrace Managed 提供的数据隐私设置

    Dynatrace Managed 群集管理员可以访问许多影响敏感最终用户数据共享的数据隐私设置。默认情况下启用所有设置。正确配置这些设置取决于组织的独特需求。除主动支持设置外，您还可以找到与新社区用户设置和域名管理相关的设置。有关可用控件的详细信息，请参阅下文。

    为确保遵守 GDPR，您必须了解 Dynatrace 捕获的个人数据，并且必须配置 Dynatrace 以保护个人数据。

    要在托管环境中配置数据隐私设置，请转至设置 > 首选项。

    注意：您必须具有群集管理员权限才能访问“ 首选项”页面。

    主动支持

    报告使用情况和结算信息

    每个 Dynatrace Managed 群集都会报告与许可证相关的消耗数据，例如每个环境的主机单元数，合成监视器和用户会话数。

    报告有关受监控技术和功能使用的信息

    Dynatrace 主动发送与您的环境相关的不兼容性或技术特定风险的警报。Dynatrace 可以报告有关已安装的 OneAgent 版本，进程技术，主机，ActiveGates 以及其他相关实体和配置的信息。检索到的信息可用于支持和改进 Dynatrace 产品。Dynatrace 可能会使用此数据（如果已汇总且无法用于识别最终用户）进行行业分析，基准测试和分析。详细了解 Dynatrace 如何在您的环境中发送有关受监控技术的信息。

    报告 Dynatrace 群集运行情况

    启用后，Dynatrace 群集将发送状态信息，包括群集 ID，隐私标志，时区，流量级别和维护时段。服务器状态（包括 CPU 核心数，CPU 负载和已用/可用存储）按群集报告。

    将报告群集和 OneAgent 事件报告给 Dynatrace 支持

    对于每个事件，群集都会发送类型，严重性级别，时间戳和描述详细信息，以便 Dynatrace 支持人员可以远程分析和解决环境中的问题或不兼容问题。禁用后，您的组织负责在联系 Dynatrace 支持之前监视系统事件并收集解决问题所需的日志文件。

    允许 Dynatrace 支持远程访问环境监视设置

    对于检测到的事件，Dynatrace 支持可以远程检查群集配置的监视设置。

    允许 Dynatrace 支持人员更改您的配置

    启用后，Dynatrace 支持可以远程优化您的环境监控设置，以确保最佳性能和稳定性。

    Dynatrace 部署健康监控

    Dynatrace Managed 安装包含一个 Dynatrace OneAgent，可以自我监控群集运行状况。

    通过从浏览器发送使用数据，帮助我们为您的用户改进 Dynatrace

    Dynatrace 使用 Woopra（一家总部位于加利福尼亚州旧金山的客户分析提供商）来分析 Dynatrace Managed 的使用情况，并为客户提供更好的体验。Woopra 使用存储在每个最终用户计算机上的 cookie。因此，捕获诸如 IP 地址，地理位置，浏览器或设备类型之类的信息以及Dynatrace Managed 内的用户动作细节。禁用此设置可防止发送此类使用情况数据。

    Dynatrace 社区设置

    登录时创建 Dynatrace 社区用户帐户

    Dynatrace 社区为客户和数字性能专家提供互联网论坛，以便彼此联系并分享想法。注册用户可以在 Dynatrace Answers 上提问并查看答案，并为Dynatrace支持创建门票。

    将 Dynatrace 帮助和答案用户论坛内容集成到产品内搜索结果中

    Managed UI 中的搜索栏可用于在 Dynatrace Answers 用户论坛和我们的[线文档中查询内容。禁用后，用户必须独立搜索这些网站以查找问题的答案。

    管理域名和 SSL 证书

    启用域名和 SSL 证书的管理

    启用此设置可dynatrace-managed.com为 Dynatrace Managed 群集生成具有受信任证书的域名（子域）。然后，您环境中的所有用户都可以访问 Dynatrace <prefix>.dynatrace-managed.com。请注意，此过程可能需要几分钟时间。完成后，您将能够访问新 URL。禁用此选项会导致 SSL 证书，并且群集 URL 将回滚到先前版本。请记住使用此 URL 更新您的 SSO IdP 设置。
