    Dynatrace Managed 硬件和系统要求

    本主题说明安装 Dynatrace Managed 的硬件和操作系统要求。

    硬件要求

    并不总是可以配置大小正确的节点，特别是如果您的环境受到不断增加的流量水平的影响。尽管对所需大小进行前期分析很有用，但如果将来监控需求增加，则能够为 Dynatrace Managed 群集添加更多容量更为重要。要充分利用Dynatrace Managed 架构的全部优势，请准备好按以下维度进行扩展：

    • 通过添加更多水平节点
    • 对于 Managed 版本1.168及更高版本，我们支持最多12个群集节点的安装。
    • 早期的 Managed 版本最多仅支持6个群集节点。
    • 垂直方向为每个节点配置更多 RAM/CPU
    • 在数据存储方面，可以根据需要调整磁盘卷的大小（有关推荐磁盘设置的指导，请参阅下文）。

    下表中包含的硬件要求是基于典型环境和负载模式的估计。个别环境的要求可能有所不同。特定列的估算考虑以下因素：

    • 最小节点规格 CPU 和 RAM 必须专用于 Dynatrace。必须禁用 CPU 的省电模式。CPU 必须以至少 2GHz 的时钟速度运行，主机应至少分配 32GB 的 RAM。

    • 事务存储 事务数据分布在所有节点上，不会冗余存储。对于多节点群集，每个节点的存储除以节点数。

    • 长期度量标准存储 对于多节点安装，我们保留度量标准存储的三个副本。对于4个或更多节点，每个节点的存储要求会降低。

    • 节点类型 监控的最大主机数（每个节点） 峰值用户 操作/分钟 （每个节点） 最小节点规格 磁盘IOPS （每个节点） 交易存储 （10天代码可见性） 长期度量标准存储 （每个节点） Elasticsearch （每个节点） （保留35天）
      使用 50 1000 4个vCPU， 32GB RAM 三十 50GB 100GB 50GB
      小 300 10000 8个vCPU， 64GB RAM 100 300GB 500GB 500GB
      中等 600 25000 16个vCPU， 128GB RAM 300 600GB 1TB 1.5TB
      大 1250 50000 32个vCPU， 256GB RAM 750 1TB 2TB 1.5TB
      超大 2500 100000 64个vCPU， 512GB RAM 1500 2TB 4TB 3TB

    例子

    • 要监控每秒峰值负载为 3k 用户操作的 8k 主机，您需要3个额外的大型节点，其中 4TB 的组合存储空间用于直接存储，30TB 用于长期指标。
    • 要监视200个主机，每秒峰值负载为500个用户操作，您需要1个中等节点，其中存储 1TB 用于事务，2.5TB 用于长期度量，或者要进行故障转移，您还可以使用3个小节点。

    存储建议

    Dynatrace Managed 存储多种类型的监视数据，具体取决于用例。 我们推荐：

    • 将 Dynatrace 二进制文件和数据存储存储在单独的装入点上，以允许独立调整数据存储大小。
    • 如果需要，在以后调整磁盘大小时，不要将 Dynatrace 数据存储在根卷上以避免额外的复杂性。
    • 在不同的磁盘卷上安装不同类型的数据存储，以实现最大的灵活性和性能。
    • 创建可调整大小的磁盘分区（例如，通过利用逻辑卷管理器[LVM]）。

    下表中包含的目录路径是默认路径。如果已安装到自定义目录，实际路径可能会有所不同。

    OneAgent 选择退出

    默认情况下启用 OneAgent 自我监控，可以选择退出安装参数：

    --install-agent <on|off>

    多节点安装

    我们建议使用多节点设置来实现故障转移和数据冗余。建议的设置是足够大小的3节点集群。对于具有多个节点的 Dynatrace Managed 安装，所有节点必须：

    • 使用相同的 Linux 发行版
    • 具有相同的硬件配置
    • 与 NTP 同步
    • 在同一时区
    • 能够通过多个端口上的专用网络进行通信
    • 节点之间的延迟应该在10毫秒左右或更短。

    系统用户

    我们建议为 Dynatrace Managed 创建的系统用户在所有节点上具有相同的UID：GID标识符。

    系统要求

    • 您需要一个专门用于 Dynatrace Managed 安装的主机。此主机不得运行 CPU 或内存密集型的其他服务，也不得运行 Dynatrace Managed 使用的其他服务。
    • 您需要64位 Linux 发行版（请参阅下面支持的 Linux 发行版）。请注意，支持物理和虚拟主机上的安装，但不支持在容器中安装。
    • Dynatrace Server 需要固定的 IP 分配。
    • 确保您已正确配置防火墙设置
    • 使用 Dynatrace Managed 安装的库可识别区域设置。要正确显示文本和符号，请确保将环境的系统区域设置设置为英语语言选项（例如，LANG=en_US.UTF-8）。

    支持的操作系统

    操作系统 版本
    Red Hat Enterprise Linux 6.x - 7.x.
    CentOS 6.x - 7.x.
    Ubuntu 12.04 - 18.x
    openSUSE 12.x - 13.x.
    SUSE Enterprise Linux 11.3 - 12.x
    Oracle Linux Server 6.x - 7.x.
    Amazon Linux AMI 2017.x - 2018.x，2.x

    必须修改 Redhat Enterprise Linux 7.4和7.5 。

    支持的文件系统

    Dynatrace Managed 可在所有常见文件系统上运行。我们建议您选择适合数据库工作负载的快速本地存储。建议不要使用 NFS 或 CIFS 等高延迟远程卷。虽然 NFS 文件系统足以用于备份目的，但我们不建议将它们用于主存储。

    亚马逊弹性文件

    系统我们不支持或推荐 Amazon Elastic File System（EFS）作为 Elasticsearch 的主存储。此类文件系统不提供 Elasticsearch 所需的行为，这可能会导致索引损坏。
