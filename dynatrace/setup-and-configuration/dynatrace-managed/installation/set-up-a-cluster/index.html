    设置群集

    设置托管群集涉及通过下载和安装 Dynatrace Managed 来创建第一个群集节点。要设置托管群集，请执行以下步骤。

    事前准备

    确保您的系统满足指定的硬件和操作系统要求。

    注意：由于可能的性能和更新稳定性问题，不建议将各种托管组件的数据存储安装到网络/远程磁盘。

    第1步：下载安装程序

    1. 登录到要安装 Dynatrace Managed 的 Linux 计算机。
    2. wget从您收到的激活电子邮件中复制命令行。
    3. 将wget命令行粘贴到终端窗口中。等待下载完成，然后继续执行步骤2。

    第2步：验证 Dynatrace Managed 安装程序签名

    Dynatrace Managed 安装程序文件经过数字签名。签名文件与 Dynatrace Managed 安装程序一起分发。结合 OpenSSL 和 Dynatrace 根证书，签名文件可用于验证安装程序的真实性。签名文件与安装程序具有相同的名称，并带有.sig扩展名。

    要验证安装程序文件，请执行电子邮件中提供的验证命令，或者：

    1. 下载 Dynatrace 根证书文件。
    2. 运行以下 OpenSSL 命令： openssl cms -verify -binary -inform PEM -CAfile dt-root.cert.pem -content dynatrace-managed.sh -in dynatrace-managed.sg.sig > /dev/null

    第3步：运行安装程序

    您需要 root 权限才能安装 Dynatrace Managed。您可以使用su或sudo运行安装脚本。为此，请在下载安装脚本的目录中键入以下命令之一。

    • Ubuntu Server
    sudo /bin/sh dynatrace-managed.sh
    • Red Hat Enterprise Linux
    su -c '/bin/sh dynatrace-managed.sh'
    • root session
    /bin/sh dynatrace-managed.sh

    如果我的公司使用 sudo 以外的权限管理系统怎么办？

    按照这些步骤，您可以通过指定用户和替换的命令来安装 Dynatrace Managed sudo。

    例如：

    /bin/sh dynatrace-managed.sh --system-user dynatrace:dynatrace --sudo-cmd  "/usr/bin/pbrun \$CMD"

    此外，出于维护目的，请考虑从以下位置添加脚本：

    • /opt/dynatrace-managed/uninstall-dynatrace.sh
    • /opt/dynatrace-managed/launcher/*
    • /opt/dynatrace-managed/utils/*

    例如，您可能希望停止在节点上运行的所有Dynatrace Managed 进程。

    pbrun /opt/dynatrace-managed/launcher/dynatrace.sh stop

    注意：请勿删除或覆盖dtrun。它是安装和更新程序所必需的。安装程序dtrun不带参数调用以验证用户是否具有管理权限，但是对于正常操作，dtrun使用参数实际运行命令的 Dynatrace 调用。

    如果您同意 Dynatrace 管理的使用条款，请键入Accept。在完成此步骤之前，安装将不会继续。要退出安装按Ctrl+C。

    Dynatrace Managed 安装程序以交互方式工作。它会显示安装路径和用户帐户等值的提示。按下接受默认值Enter。要覆盖设置，请在终端中键入您的选择，然后按Enter。

    注意：请提供您的Dynatrace Managed 许可证密钥。没有许可证密钥，您无法完成安装。

    • 什么时默认设置？

    • 安装路径（二进制文件）： /opt/dynatrace-managed

    • Dynatrace Server 数据文件： /var/opt/dynatrace-managed
    • 运行 Dynatrace 进程的系统用户： dynatrace

    • 运行 Dynatrace 进程的系统组： dynatrace

    • 我可以绕过交互模式吗？

    要绕过所有问题并使用默认设置进行安装，请使用--install-silent参数运行安装程序。请务必提供 Dynatrace Managed 许可证密钥作为--license参数值。

    • 安装期间会发生什么？

    Dynatrace Managed 群集是一组专用组件，已配置为托管监视环境和进程监视数据。

    安装程序在安装目录中设置以下组件（默认情况下/opt/dynatrace-managed）：

    • 预配置的 Java 运行时环境（您的操作系统设置不受此影响）。在您的alternatives选项中不可见。
    • 基于 Cassandra 的 Hypercube 存储
    • 基于弹性搜索的搜索引擎
    • Dynatrace 服务器
    • 嵌入式 ActiveGate

    安装程序还优化了操作系统设置：

    • 交换已关闭（带swapoff）。 请注意，启用交换可能会导致意外行为，因此不受支持。
    • iptables“ PREROUTING”规则得到增强，可以将通信转发到 Dynatrace Server（通过端口8021上的 HTTPS）。要查看确切的规则，请键入iptables -L -vt nat终端。
    • readahead 页面缓存设置为512。
    • 全局更改用户的限制（无限制的锁定内存地址空间，无限的地址空间，增加的进程数和打开文件限制）。详情/etc/security/limits.conf请见。
    • max_map_count 被修改了。

    在安装 Dynatrace Managed 期间，可能会修改以下系统文件和目录：

    /etc/hosts` `/etc/sysctl.conf` `/etc/pam.d/su` `/etc/rc.local` `/etc/security/limits.conf` `/etc/security/limits.d/90-nproc.conf` `/etc/sudoers` `/etc/sudoers.d/` `/etc/init.d/` `/etc/init.d/rc*.d/` `/etc/systemd/system/

    • 我在哪里可以找到安装日志？

    您可以在/opt/dynatrace-managed/installer/目录中找到 Dynatrace Managed 安装日志文件。要轻松识别正确的日志文件，请在文件名中查找安装日期。例如，9月30日成功安装的日志文件将是20160930-173309-success-install-of-managed-installer.log

    注意：要查看所有可用安装参数列表，请使用参数运行安装程序--help

    第4步：完成设置

    1. 复制安装过程结束时提供的环境 URL，将地址粘贴到浏览器中以完成剩余的安装，出现以下页面。

    1. 为您的第一个监控环境提供名称并设置管理员帐户。完成后，单击“ 下一步”。您将登录并定向到 Cluster Management Console。生成默认域，应使用该域代替安全（HTTPS）连接的 IP 地址。

    1. 从Cluster Management Console 菜单中，单击“ 环境”以查看新创建的环境。选择您的环境，然后单击“ 转到环境”以访问环境的Web UI。您可以稍后创建其他环境。
