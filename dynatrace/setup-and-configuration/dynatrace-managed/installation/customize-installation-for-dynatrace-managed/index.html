    自定义安装 Dynatrace Managed

    您可以使用命令行参数覆盖某些默认设置或升级 Dynatrace Managed。

    默认安装

    默认情况下，您无需使用任何参数来安装 Dynatrace Managed。安装程序以交互模式工作。它要求您确认默认设置或提供自己的值。您在安装期间输入的值始终优先。它们会覆盖命令行参数值以及默认值。

    使用命令行参数以交互模式运行安装程序时，参数值将显示为提示（而不是默认值）。如果此时输入任何值，它们将优先。

    要使用默认设置在非交互模式下安装Dynatrace Managed，请使用该--install-silent 参数。请记住使用--license 提供我们发送给您的许可证密钥。

    • 什么是默认设置？
    • 安装路径（二进制文件）： /opt/dynatrace-managed
    • Dynatrace Server数据文件： /var/opt/dynatrace-managed
    • 运行Dynatrace进程的系统用户： dynatrace
    • 运行Dynatrace进程的系统组： dynatrace

    命令行参数

    请参阅下面 Dynatrace Managed 安装程序支持的命令行参数。

    • 初始参数配置

    应使用以下环境和 admin-user 属性参数进行初始服务器配置。指定后，安装程序将在成功安装和群集注册后为公共 REST API 生成身份验证令牌。

    --initial-environment <name> - 使用此参数指定环境的名称。

    --initial-first-name <name> - 使用此参数指定管理员的名字。

    --initial-last-name <name> - 使用此参数指定管理员的姓氏。

    --initial-email <email> - 使用此参数指定管理员的电子邮件。

    --initial-pass <pass> - 使用此参数指定管理员密码。

    • 安装方式

    您可以使用以下任何模式运行 Dynatrace Managed 安装程序：

    --install - 这是标准的交互式安装。

    --install-silent - 使用此参数以默认设置进行安装。您可以使用此参数自动执行 Dynatrace Managed 安装（无提示）。您可以使用其他命令行参数来绕过某些默认设置。

    --upgrade - 开始升级过程。如果要从命令行启动 Dynatrace Managed 升级，请使用此参数。

    --uninstall - 安全卸载 Dynatrace Managed。

    --restore- 从--backup-file参数引用的备份文件恢复 Dynatrace Managed 安装。一旦在 Dynatrace Managed 上启用并配置了备份，Dynatrace Managed 每天都会执行备份。

    --system-check - 使用此参数检查机器和操作系统是否满足新版本的要求。

    --self-check - 使用此参数验证安装程序文件是否已损坏。

    • 设置参数

    您可以使用以下参数更改安装路径和其他设置：

    --license <license-key> - 使用此参数提供从 Dynatrace 团队获得的授权密钥。

    --license-file <path> - 未提供许可证密钥时使用的 Dynatrace 授权文件。

    --binaries-dir <path> - Dynatrace 二进制文件目录的完整路径。

    --datastore-dir <path> - Dynatrace 安装空间目录的完整路径。所有 Dynatrace 数据都保存在此处。建议您为此目录使用专用驱动器或分区。

    --cas-datastore-dir <path> - Dynatrace 指标存储库目录的完整路径。如果您指定此位置，则会将指标数据保留在此处而不是主数据位置。

    --els-datastore-dir <path> - Dynatrace Elasticsearch 商店目录的完整路径。

    --svr-datastore-dir <path> - Dynatrace 原始事务存储目录的完整路径。如果指定此位置，则原始事务数据将保留在此处而不是主数据位置。

    --system-user <user:group>- name:group有权运行 Dynatrace 进程的系统用户。

    --network-proxy- 如果您的计算机使用网络代理连接到 Internet，请按以下格式将地址放在此处：protocol://[user:password@]server-address:port。默认值为none。

    --cluster-ip - 如果您的计算机有多个网络接口，则需要确定哪个网络接口将用于 Dynatrace 群集流量并将其IP4地址放在此处。

    --install-agent <on|off> - 启用/禁用自我监控 OneAgent 的安装。默认值为on。

    --agent-dir <path> - 允许将自我监控 OneAgent 安装到其他目录。例如，在 Linux 上：/bin/sh Managed-installer.sh --agent-dir /data/dynatrace/。

    与--agent-dir设置为 /data/dynatrace/，安装程序创建的符号链接/opt/dynatrace/oneagent- > /data/dynatrace和所有 OneAgent 文件放置到指定的目录（在本例中，/data/dynatrace）。请注意，卸载 OneAgent 后，需要手动删除此符号链接。启用 SELinux 时在 Linux 上使用此参数需要在semanage系统上使用二进制文件。

    --hosts <on|off> - 启用/禁用 / etc / hosts 文件的更改。默认是on。

    --generate-domain <on|off> - 生成域名和相应的证书。默认值为on。

    --no-start - （仅限升级）如果不希望 Dynatrace Managed 在升级后立即启动，请使用此参数。

    --unregister - 卸载群集时，请使用此参数以取消注册群集并从 Mission Control 中释放许可证。

    如果要使用安装文件在群集中安装辅助 Dynatrace Managed 节点，请使用以下参数：

    --seed-ip <ip> - 群集中主服务器的 IPv4 地址。

    --seed-auth <auth-token> - 用于连接到种子节点的身份验证令牌。您可以在下载节点安装程序页面上登录种子节点后找到此信息。

    • SSL 证书

    命令行参数还可以帮助您在 Dynatrace Managed 上安装或更新SSL证书。有关安装完全合格的数字证书的详细信息，请参阅我可以使用自己的SSL证书吗？

    --update-cert - 如果要替换在 Dynatrace Managed 安装期间生成的自签名证书，请将此参数与安装脚本一起使用，并安装 CA 颁发的证书。您需要与此参数一起使用--ssl-keystore。

    --ssl-keystore <path> - 使用此参数为安装脚本提供包含CA颁发的证书的KeyStore文件的路径。

    • 杂项

    --version - 打印版本信息。

    --tmp-dir <path> - 安装程序临时文件目录的完整路径。

    列出可用选项

    您始终可以使用该help 参数列出 Dynatrace Managed 安装程序的所有可用命令行选项。以root身份输入：

    ./dynatrace-managed-installer.sh --help
