    安装自己的SSL证书

    Dynatrace UI 只能通过加密的 HTTPS 连接访问。要确保安全访问并避免浏览器警告，您必须配置有效的 SSL 证书。Dynatrace 可以自动为您管理 - 每个集群都有一个专用的 Web 域（子域dynatrace-managed.com）和一个可信的 SSL 证书。您可以使用此域访问 Dynatrace UI 而不接收浏览器警告。

    如果您不希望 Dynatrace 为您创建域和 SSL 证书，请转到 Cluster Management Console 中的 Settings > Preferences 并禁用 Manage domain name 和 SSL certificates 选项。然后，Dynatrace 将使用自签名 SSL 证书。默认情况下不会信任自签名证书 - 第一次尝试连接到 Dynatrace Managed 时，您将收到安全警告。在浏览器安全设置中确认此例外。

    如果要使用自己的可信 SSL 证书，请在禁用自动管理后按照以下说明操作。

    事前准备

    您需要

    • 您的 SSL 证书以及从证书颁发机构（CA）收到的密钥文件：

    • 服务器证书（.cer 或.cert）

    • 根证书和中级证书（.cer 或.cert）

    • 证书的私钥（.pem）

      加密的私钥

      • 我们不支持加密的私钥。要解密 SSL 私钥，请运行以下命令： openssl rsa -in encrypted.ssl.key -out decrypted.ssl.key
      • 替换encrypted.ssl.key为加密的 SSL 私钥的文件名以及解密的 SSL 私钥decrypted.ssl.key的输出文件。该命令将提示您输入密码并将解密的密钥保存在文件中decrypted.ssl.key。

    • （可选）对于命令行安装：

    • Dynatrace Managed 安装脚本

    • OpenSSL 工具包

    在 Dynatrace Server 上安装可信证书

    如果要使用自己的证书或 CA 颁发的证书，请将证书上载或粘贴到 Dynatrace Server。您还可以将与证书关联的主机名设置为 Dynatrace Server 配置的一部分。

    1. 以管理员身份登录 Dynatrace Server。

    2. 在 Dynatrace Managed 部署状态页面上，选择需要新证书的群集节点。

    3. 在“ 节点详细信息”页上，单击“ 编辑 SSL 证书”。

    4. 您可以粘贴或上载从 CA 颁发机构收到的密钥文件。

    5. 您的私钥在私钥框中。

    6. 您的公钥证书框中的服务器证书。
    7. 证书链中的根证书和中间证书。

    粘贴密钥时，请确保在文本字段中包含页眉和页脚。

    1. 单击“ 保存” 以上载证书。

    名称不匹配错误

    您的证书与特定主机名相关联。要避免名称不匹配错误，请确保 SSL 证书中的公用名（域名）与浏览器地址栏中的地址匹配。

    在 Dynatrace Managed 安装期间安装证书

    您需要做的就是使 Dynatrace Managed 安装脚本可以访问 KeyStore 文件。

    在 Dynatrace Managed 安装期间，您可以使用该--ssl-keystore参数将安装程序指向 PKCS12 KeyStore 所在的位置。然后，安装程序将使用 KeyStore 而不是生成自签名证书。

    我如何准备 KeyStore

    您需要将服务器证书和私钥组合到 PKCS12 SSL KeyStore 中。使用 OpenSSL 生成。在命令行中，确保使用dynatraceserver名称值和dynatracepass 值：

    openssl pkcs12 -export -out <dynatrace-keystore.pkcs12> -name dynatraceserver -password pass:dynatrace -in <server_certificate.cer> -certfile <root-and-intermediate-certificates.cer> -inkey <private-key-for-certificates.pem>

    请注意，为确保 Dynatrace Server 能够正确识别证书，-name dynatraceserver -password pass:dynatrace请勿更改。

    如果您打算在 Dynatrace Managed 安装期间或稍后安装证书，则需要将KeyStore保留在计算机上。

    示例

    如果您以 root 用户身份登录并希望/tmp/mycomp-ssl-cert.pkcs12在安装期间使用，请使用以下命令安装 Dynatrace Managed 和 CA 颁发的证书：

    dynatrace-managed-installer.sh --install --ssl-keystore /tmp/mycomp-ssl-cert.pkcs12 --license 1234abc567

    请注意，您需要提供 KeyStore 文件的完整路径作为--ssl-keystore参数值。
