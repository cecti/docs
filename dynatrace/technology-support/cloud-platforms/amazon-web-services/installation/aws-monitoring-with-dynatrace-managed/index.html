    为 AWS 设置 Dynatrace Managed 监控

    您可以将 Dynatrace 与 Amazon Web Services（AWS）集成，以智能监控 Amazon Cloud 中运行的服务。AWS 集成可帮助您掌握云中数据中心的动态。

    重要：

    为了成功将 Dynatrace 与 AWS 集成以监控服务，必须在 AWS 中托管至少一个 Dynatrace 组件。可以使用或不使用 Environment ActiveGate 部署 Dynatrace 。确保满足以下部署要求之一：

    • 对于使用 Environment ActiveGate 的部署，必须在 AWS 中托管 Environment ActiveGate。
    • 对于没有 Environment ActiveGate 的部署，必须在 AWS 中托管 Dynatrace Managed 服务器。

    概述

    按照以下基本步骤将 Dynatrace Managed 与 Amazon Web Services（AWS）集成：

    1. 创建 AWS 监控策略
    2. 选择一种访问方式：
    3. 基于密钥的访问
    4. 基于角色的访问
    5. 定义 AWS 资源标记

    成本

    Dynatrace 每5分钟发出一次 Amazon API 请求。除了 CloudWatch API 调用之外，Dynatrace 还会对受监控的 AWS 服务进行 API 调用，以了解其实例，标记等。下面的“ 创建监控策略”部分中提供了被调用服务和操作的列表。以下是 AWS 监控成本的粗略估计：

    AWS 服务 指标数量 每个实例的每日费用（美元）
    弹性计算云（EC2） 7 $ 0.02016
    弹性块存储（EBS） 8 $ 0.02304
    弹性负载平衡器（ELB） 11 $ 0.03168
    关系数据库服务（RDS） 11 $ 0.03168
    DynamoDB 15 $ 0.06912
    LAMBDA 4 $ 0.01152

    亚马逊收费

    亚马逊将从 CloudWatch API 请求的每1,000个指标收取约0.01美元的费用，并在您使用 Dynatrace 的 AWS 账户的账单中包含费用。

    AWS 监控策略

    AWS 监控策略定义了您需要为 Dynatrace 监控 AWS 账户中运行的服务所需的最小权限范围。创建一次并在启用 Dynatrace 访问您的 AWS 账户时随时使用。

    1. 转到 Amazon 控制台中的身份和访问管理（IAM）

    2. 转到“策略”并单击“创建策略”

    3. 选择 JSON 选项卡，然后粘贴下面的框中的预定义策略。

    4. JSON 中的预定义策略

      { "Version": "2012-10-17", "Statement": [ { "Effect": "Allow", "Action": [ "autoscaling:DescribeAutoScalingGroups", "cloudwatch:GetMetricData", "cloudwatch:GetMetricStatistics", "ec2:DescribeAvailabilityZones", "ec2:DescribeInstances", "ec2:DescribeVolumes", "elasticloadbalancing:DescribeLoadBalancers", "elasticloadbalancing:DescribeTags", "elasticloadbalancing:DescribeInstanceHealth", "elasticloadbalancing:DescribeListeners", "elasticloadbalancing:DescribeRules", "elasticloadbalancing:DescribeTargetHealth", "rds:DescribeDBInstances", "rds:DescribeEvents", "rds:ListTagsForResource", "dynamodb:ListTables", "dynamodb:ListTagsOfResource", "lambda:ListFunctions", "lambda:ListTags", "elasticbeanstalk:DescribeEnvironments", "elasticbeanstalk:DescribeEnvironmentResources", "s3:ListAllMyBuckets", "sts:GetCallerIdentity", "cloudformation:ListStackResources", "tag:GetResources", "tag:GetTagKeys" ], "Resource": "*" } ] }

    5. 为该政策命名。例如Dynatrace_monitoring_policy。在“ 名称”字段中键入它。

    6. 单机“创建策略”按钮

    访问方法

    要获得全面 AWS 云计算监控所需的信息，Dynatrace 需要识别 AWS 环境中的所有虚拟化基础架构组件，并收集与这些组件相关的性能指标。我们使用此信息来了解应用程序，服务和主机的上下文。为此，您需要授权 Dynatrace 访问您的 Amazon 指标。

    您可以通过使用专用访问密钥（基于密钥的访问）或为 Dynatrace 定义特殊角色（基于角色的访问）来启用对 AWS 度量标准的 Dynatrace 访问。在所有情况下，请确保您的 Environment ActiveGate 或 Managed 服务器与 AWS 具有有效连接。在 Managed 或 ActiveGate 中配置代理，或*.amazonaws.com在防火墙设置中配置白名单。

    作为最佳实践，使用临时安全凭据（IAM 角色）代替访问密钥，并禁用任何AWS帐户根用户访问密钥。

    基于密钥的访问

    如果您决定使用基于密钥的身份验证，请记住定期轮换密钥。请记住，每次更改密钥时都需要执行此过程。

    您需要：

    • 创建新 AWS 用户的权力
    • 您的 AWS 账户 ID
    • 您的 Amazon Access 密钥 ID 和秘密访问密钥

    使用基于密钥的访问启用对 Amazon 帐户的访问

    Dynatrace 可以使用访问密钥对 AWS 服务 API 发出安全的 REST 或查询协议请求。您需要生成一个 Access 密钥 ID和一个 Secret 访问密钥，Dynatrace 可以使用它来从 Amazon Web Services 获取指标。

    1. 在 Amazon 控制台中，转到“ 用户”，然后单击“ 添加用户”。

    2. 输入要创建的密钥的名称（例如，Dynatrace_monitoring_user）。在 Select AWS access type 中，选择 Programmatic access 选项，然后单击 Next：Permissions 按钮。

    3. 点击直接附加现有政策，并选择您定义的监控策略，例如Dynatrace_monitoring_policy。单击下一步：查看。

    4. 查看用户详细信息，然后单击“ 创建用户”按钮。

    5. 存储访问密钥 ID 名称（AKID）和密钥访问密钥值。 您可以下载用户凭据或复制在线显示的凭据（单击“ 显示”）。

    使用基于密钥的访问将您的 Amazon 账户连接到 Dynatrace

    一旦您授予了对 Dynatrace 的 AWS 访问权限，就可以将 Dynatrace 连接到您的 Amazon AWS 账户了。

    1. 在 Dynatrace 中，转至设置>云和虚拟化> AWS，然后单击连接新实例。

    2. 选择基于密钥的验证方法。

    3. 为此连接创建名称。这是强制性的。Dynatrace 需要此名称来标识和显示连接。
    4. 在“ 访问密钥 ID” 字段中，粘贴您在 Amazon 中为 Dynatrace 访问创建的密钥的标识符。
    5. 在“ 密钥访问密钥” 字段中，粘贴您在 Amazon 中为 Dynatrace 访问创建的密钥的值。

    6. 单击“ 连接” 以验证并保存连接。

    7. 成功验证并保存连接后，您的 AWS 账户将列在云和虚拟化设置页面中。您应该很快就会看到 AWS 云监控数据。

    基于角色的访问

    在典型设置中，您将创建并附加两个角色，一个用于 Dynatrace，另一个用于 AWS 基础架构中托管的 Environment ActiveGate（或 Dynatrace Managed 服务器）。

    您需要：

    • Environment ActiveGate 或 Dynatrace Managed 服务器（适用于没有 Environment ActiveGate 的部署）安装在 AWS EC2 主机上。他们必须能够在您的 AWS 账户中担任一个角色，允许其读取 Dynatrace 监控数据。
    • 您的 Amazon Web Services 源帐户 ID，即托管 Dynatrace 组件的帐户。
    • 启动 Dynatrace Managed 服务器或 Environment ActiveGate的角色的名称。
    • Amazon Web Services 目标帐户 ID，即您要监控的帐户。
    • 从设置 > 云和虚拟化 > AWS复制外部 ID。

    使用基于角色的访问权限启用对 Amazon 账户的访问权限

    下面描述的步骤适用于源帐户和目标帐户相同以及帐户不同的情况。如果要监视多个目标帐户，请对每个帐户重复步骤1，然后将其添加到Statement.Resource步骤2.4中策略中的阵列。

    步骤1.在目标账户上为 Dynatrace 创建监视角色

    1. 在浏览器中，打开一个新选项卡并登录 Dynatrace 以获取外部 ID。转至设置 > 云和虚拟化 > AWS，单击连接新实例，选择基于角色的身份验证方法，然后单击令牌字段旁边的复制。
    2. 转到 Amazon 控制台中的身份和访问管理（IAM）。
    3. 转到角色，然后单击“ 创建角色”。
    4. 选择另一个 AWS 账户磁贴并使用 Dynatrace 帐户建立信任。粘贴您在第一步中复制的外部 ID。粘贴用于访问目标帐户的12位源帐户 ID。单击“ 下一步：权限”。
    5. 在附加权限策略部分，选择您所创建的监控策略，例如Dynatrace_monitoring_policy。单击下一步：查看。
    6. 例如，在 Review 页面上，提供角色名称Dynatrace_monitoring_role。请记住，稍后您需要将您的亚马逊帐户连接到 Dynatrace。单击“ 创建角色”按钮。

    步骤2.在源账户上为 ActiveGate 或Dynatrace Managed 服务器主机创建角色

    在目标帐户上创建Dynatrace_monitoring_role之后，为 Environment ActiveGate 或 Dyntrace Managed 服务器创建一个角色，该角色将负责 AWS 监控。

    1. 在 Amazon 控制台中，转到角色，然后单击创建角色。选择 AWS 服务和 EC2 作为将使用该角色的服务。单击下一步：权限，然后跳到查看页面。

    2. 在 Review 页面上，例如为角色提供名称，Dynatrace_ActiveGate_role然后单击 Create role。

    3. 选择Dynatrace_ActiveGate_role并单击“ 添加内联策略”

    4. 选择 JSON 并粘贴下面的预定义策略。编辑并添加：

    5. 12位数的目标帐号

    6. 在前面的步骤（例如Dynatrace_monitoring_role）中创建的角色名称，用于承担监视角色。

    不要包含<and>字符。

    • Dynatrace 监视角色预定义策略

      { "Statement": [ { "Resource": [ "arn:aws:iam::<12-digit target account number>:role/<role created in previous step (Dynatrace_monitoring_role)>" ], "Action": [ "sts:AssumeRole" ], "Effect": "Allow" } ], "Version": "2012-10-17" }

      完成后，粘贴更新策略，然后单击查看策略。

    • 例如，为刚刚创建的内联策略命名Dynatrace_assume_policy，然后单击“ 创建策略”。

    • 返回Dynatrace_monitoring_role之前创建的并在角色摘要页面中选择 Trust relationships 选项卡，然后单击 Edit Trust Relationships。

    • 将下面的 JSON 示例粘贴到文本编辑器中。编辑并添加：

    • 12位数的来源帐号

    • 启动 Environment ActiveGate 或 Dynatrace Managed 服务器的角色，以及在先前步骤中复制的外部标识。

    不要包含<and>字符

    • Dynatrace 假设策略

      { "Version": "2012-10-17", "Statement": [ { "Effect": "Allow", "Action": "sts:AssumeRole", "Principal": { "AWS": [ "arn:aws:iam::<12-digit source account number>:role/<role for your Dynatrace Managed Server (Dynatrace_ActiveGate_role)>", ] }, "Condition": { "StringEquals": { "sts:ExternalId": "<External ID generated from tenant>" } } } ] }

      粘贴已修改的文本并单击“ 更新信任策略”按钮。

    • 转到 EC2 控制台，右键单击托管 Environment ActiveGate 或 Dynatrace Managed 服务器的实例，然后选择“ 实例设置” >“ 附加/替换 IAM 角色”。

    • 选择之前创建的角色Dynatrace_ActiveGate_role，然后单击“ 应用”。

    步骤3.修改 ActiveGate 配置

    1. 为您的 Environment ActiveGate 编辑custom.properties的文件。

    2. 设置以下属性如下：

    ``` [vertical.topology] use_aws_proxy_role = false

    [aws_monitoring] aws_monitoring_enabled = true ```

    只能有一个：

    • 如果部署中有多个 ActiveGates，请将该aws_monitoring_enabled属性设置为true仅在一个 ActiveGate 上。所有其他 ActiveGates 都应将此属性设置为false。
    • 请记住，Dynatrace 集群节点包含嵌入的 ActiveGates。确保将这些 ActiveGates 的aws_monitoring_enabled属性设置为false。

    如果 ActiveGate 专用于 AWS 监控，则还必须将该MSGrouter属性设置为false：

    [collector] MSGrouter = false

    删除aws_proxy_account和aws_proxy_role属性。

    1. 保存文件并重新启动 ActiveGate。

    使用基于角色的访问将您的 Amazon 账户连接到 Dynatrace

    一旦您授予了对 Dynatrace 的 AWS 访问权限，就可以将 Dynatrace 连接到您的 Amazon AWS 账户了。

    1. 在 Dynatrace 中，转至设置>云和虚拟化> AWS，然后单击连接新实例。
    2. 选择基于角色的身份验证方法。
    3. 为此连接创建名称。如果将此字段留空，则将在 Dynatrace 页面上使用名称 Role 来定义此连接。
    4. 例如，在“ 角色” 字段中，键入您在 Amazon 中为 Dynatrace 创建的角色的名称Dynatrace_monitoring_role。
    5. 输入您的帐户 ID （您希望我们从中提取指标的帐户）。
    6. 单击“ 连接” 以验证并保存连接。
    7. 成功验证并保存连接后，您的 AWS 账户将列在云和虚拟化设置页面中。 您应该很快就会看到 AWS 云监控数据。

    定义 AWS 资源标记

    我们建议限制 AWS 监控的范围，并减少对 Amazon 的 API 调用次数。您可以使用标记来限制 Dynatrace 监视的 AWS 资源。有关详细信息，请参阅如何标记 AWS 资源？
