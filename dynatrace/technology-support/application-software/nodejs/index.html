    Node.js

    Node.js 是基于 Google 的 V8 JavaScript 引擎的服务器端框架。 Node.js 具有异步执行模型，经常用于粘合或作为企业环境中的代理层。

    功能

    Dynatrace 提供广泛的 Node.js 监控功能：

    • 堆和流程指标
    • 堆转储
    • CPU 采样（参见已知限制）
    • 事件循环指标
    • 深入了解入站和出站 HTTP 调用
    • 专门支持各种数据库（包括查询捕获）
    • OneAgent SDK 用于自定义跟踪

    有关与 Node.js 一起使用的受支持技术的详细信息，请参阅我们支持的技术矩阵。

    支持和支持

    Node.js 遵循 LTS 版本模型。

    每个新的偶数版本发布后，每个奇数版本都会很快到达 EOL。每个偶数版本最终都会成为LTS版本。对于企业生产环境，我们建议您坚持使用 LTS 版本。

    每当发布一个新的 Node.js 主要版本（偶数或非偶数）时，我们就会添加对该版本的支持。

    Dynatrace 将遵循此支持模型，但将支持每个 Node.js 版本至少半年，以便为我们的客户提供升级时间。

    已知限制

    • 由于 JavaScript 和 Node.js 的平台限制，与 .NET 或 Java 相比，代码级可见性有限。
    • 结合不受支持的第三方模块，异步回调中的上下文可能会丢失。在这种情况下，请通过单击Dynatrace 菜单栏右上角的聊天按钮联系 Dynatrace ONE 产品专家。
    • Google V8 受到一系列内存和 CPU 开销问题的影响，这是生产级后台 CPU 分析的阻止程序。因此，将来将禁用 CPU 分析。目前，如果您遇到此问题，请在深度监控设置中将其禁用。
    • 内存泄漏
    • 过度的峰值内存消耗
    • Windows 系统上的 CPU 开销过高
    • 默认情况下，从 OneAgent 版本 1.157 开始禁用环境采样 由于零星的崩溃，不支持 Node.js 版本 4。

    Linux： 可以为以下 Node.js 版本启用环境采样： * Node.js 版本 6 * Node.js 版本^ 8.9.4（8.9.4 <= node.js <9.0.0） * Node.js版本 9.3.0 或更高版本

    Windows： 默认情况下禁用环境采样，因为它会导致一个 CPU 100％ 的 CPU 使用率。

      OneAgent 会为其检测到的每个不受支持的版本记录日志条目：
  AmbientSampling: <unsupported platform> / <node version>
