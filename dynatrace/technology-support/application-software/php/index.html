    PHP

    PHP 是一种服务器端脚本语言，特别适用于 Web 开发，但在通用编程中也很流行。最初由 Rasmus Lerdorf 于 1994 年创建，新版本现在由 The PHP Group 制作。

    PHP 由以下 PHP 解释器处理：

    • Apache Web 服务器
    • 通用网关接口（CGI）可执行文件
    • 命令行界面（CLI）

    在 Web 服务器上运行 PHP 的最常用方法是：

    • mod_PHP - Apache HTTP 服务器上的模块。PHP 作为 Apache 模块运行，这意味着 PHP 请求在 Apache 进程下运行，这使得 PHP 运行良好，但灵活性受到 Apache 配置，权限，限制等的限制。

    • PHP_FPM - FastCGI Process Manager (FPM)使用池管理来优化性能。每个池充当一个独立的 PHP 实例，使您能够为每个池创建一个独立的配置，这提供了粒度控制和更大的灵活性。这是运行 PHP 最快的方法。

    功能

    Dynatrace 提供广泛的 PHP 监控功能：

    • 所有数据库语句和 SQL 指标
    • 编译，执行和响应时间分析
    • 详细的请求和响应指标
    • 有关重新启动，崩溃和部署更改的信息
    • 深入了解堆栈问题（如堆栈溢出）
    • 自动收集 PHP-FPM 指标
    • 代码级别的热点位置
    • 通过 CURL、SOAP 接口和其他远程接口(如fopen或get_file_contents)分析对外部服务的请求

    请参阅 Dynatrace 支持哪些环境和版本？有关与 PHP 结合使用的受支持技术的详细信息。

    Windows 和 Linux 支持 PHP 监视。

    支持与否

    不同的 PHP 版本具有 PHP 定义的支持时间轴。请参阅 php.net 上支持的 PHP 版本，了解 PHP 当前支持哪些版本以及何时计划对每个版本的结束支持。

    • 不推荐使用的版本不会收到更新或安全补丁，因此应避免在生产环境中运行它们。
    • Dynatrace 承诺支持每个 PHP 版本至少到供应商不再支持为止(在大多数情况下，支持至少在此之后延长6个月)。

    当 OneAgent 检测到不受支持的PHP版本时，我们会将其从监视中排除，并在流程概述页面中显示警告：

    Activation of deep monitoring was unsuccessful
Process version is not supported

    我们还将消息写入主机上的错误日志。

    已知的限制

    • 不支持 CLI 监视，因为在短暂的 CLI 执行期间，注入开销可能占整个运行时的很大一部分。
    • CGI 和 Fast-CGI 之间没有区别; 这些报告为统一类型：CGI。
    • 我们强烈建议不要使用诸如 system、exec、proc_open 或 shell_exec 之类的命令调用 PHP CLI 应用程序，这可能会导致很大的开销。
