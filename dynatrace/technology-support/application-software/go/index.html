    Go

    Go 是由 Robert Griesemer，Rob Pike 和 Ken Thompson 创建的编程语言。Go 是许多组织首选的云原生编程语言。

    功能

    • Go 可执行程序的自动注入和插装
    • 不间断 24 x7 生产级 CPU 性能分析
    • 特定于 Go 的指标：
    • 暂停时间
    • Committed，Used，Idle 和 Live 堆内存大小
    • Goroutine 计数（应用/系统）
    • Go 托管内存堆:堆外、堆栈和总体提交/使用的内存
    • 分配的 Go 对象计数
    • 垃圾收集器调用计数
    • Go 运行时系统调用计数，cgo 调用计数
    • 全局 Goroutine 运行队列大小
    • 已停止、未工作，以及整个工作线程计数、空闲调度上下文计数
    • 传入/传出 Web 请求监视
    • 定制服务监控

    支持与否

    Go release 策略支持最后两个主要的 Go 版本。

    每当发布新的 Golang 主要版本（均匀或不均匀）时，我们都会添加对该版本的支持。Dynatrace 将为每个次要版本添加支持; 您可以查看版本矩阵以获取更多详细信息。

    Dynatrace 将遵循这一支持模式，将支持每个 Go 版本至少半年，以给我们的客户提供升级的时间。

    已知的限制

    仅支持官方，稳定的 Go 版本

    Go 支持仅限于使用 golang 工具链编译的官方稳定 Go 版本。

    OneAgent 不支持使用gccgo工具链编译的二进制文件。

    应用二进制文件必须动态链接

    此限制仅适用于 Linux 系统。

    OneAgent 全自动注入需要动态链接应用程序的二进制文件。如果应用程序使用某些标准运行时库包（net/http），则会自动应用动态链接。

    在所有其他情况下，可以使用命令行选项强制执行动态链接-ldflags '-linkmode=external'。

    例

    以下是极简的 Go 应用（GoMinimal.go）

    package main
import "fmt"

func main() {
    fmt.Print("Enter text: ")
    var input string
    fmt.Scanln(&input)
    fmt.Print(input)
}

    构建应用程序将生成静态链接的应用程序二进制文件：

    $ go build GoMinimal.go
$ file GoMinimal
GoMinimal: ELF 64-bit LSB executable, x86-64, version 1 (SYSV), statically linked, n

    实施动态链接-ldflags '-linkmode=external'：

    $ go build -ldflags '-linkmode=external' GoMinimal.go
$ file GoMinimal
GoMinimal: ELF 64-bit LSB executable, x86-64, version 1 (SYSV), dynamically lin

    无法使用-linkshared选项构建应用程序

    Go 支持 Go 标准库的动态链接。很少使用此构建模式，OneAgent 不会注入以这种方式构建的应用程序。

    样本申请 -linkshared

    $ go install -buildmode=shared -linkshared std
$ go build -linkshared GoMinimal.go

    生成的应用程序二进制文件将被 OneAgent 拒绝。

    不支持加载 Go 插件的应用程序

    Go 插件是使用 build 标志-buildmode=plugin编译的包，用于生成共享对象文件。这种构建模式很少使用，当应用程序实际加载 Go 插件时，OneAgent 将禁用深度监视。

    应用程序必须包含符号表

    OneAgent 依赖于存储在应用程序二进制文件符号表中的信息。默认情况下，Go 会在应用程序二进制文件中生成一个符号表，但这可以通过命令行参数或外部工具（如strip）加以抑制。

    go run

    很少使用的go run <application>命令可以动态构建和运行应用程序。由于输出应用程序文件是临时的（在应用程序终止后自动删除），因此生成的应用程序二进制文件没有符号表。因此，OneAgent 无法监视生成的应用程序。

    支持musl libc

    musl libc 库是 glibc 的替代品。从 OneAgent 1.167 版本开始，Dynatrace 支持基于 musl libc 的 Go 应用程序（例如 Alpine Linux）。

    虽然 musl libc 非常类似于 glibc 功能，但两者之间存在微妙的行为差异。此外，Go 没有正式支持基于 musl 的 Go 工具链，这意味着无法从 golang.org 网站下载 Go 工具链二进制文件。

    此外，Go 使用 musl libc 存在严重问题。这限制了 Dynatrace 可以支持基于 musl 的应用程序的程度。Go 工具链包含一个内部链接器，用于生成基于 musl 的应用程序二进制文件，这些二进制文件在应用程序启动时无法正确初始化 musl libc。此问题会阻止 Dynatrace 监视这些应用程序。Dynatrace 将在相关的应用程序进程页面上显示消息“深度监视的激活不成功，不支持使用 Go 内部链接器构建的 Go musl 二进制文件的监视”。

    使用系统链接器生成应用程序二进制文件将添加正确初始化共享对象的启动代码。添加-ldflags '-linkmode external'到构建命令行会强制使用系统链接器。生成的二进制文件可以由 Dynatrace 监视，并将使用正确初始化的 libc 执行。
