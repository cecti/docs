    创建自定义 Java 服务

    在创建自定义 Java 服务之前，请确保您的系统满足以下先决条件：

    • Dynatrace 版本 1.144
    • OneAgent 版本 1.133

    如何创建自定义 Java 服务

    考虑创建一个定制的服务,它是基于一个特定组件的化身。在这个例子中，我们为 EasyTravel 应用程序创建了一个名为 Booking 的示例服务:

    1. 选择要监视的所有方法。这些方法必须存在于所有组件中。只要所有类中都存在相同的方法签名，就可以使用基类或接口来选择这些方法。

    2. 转至设置 > 服务器端服务监控 > 自定义服务检测。

    3. 在“ Java 服务”选项卡下，单击“ 定义 Java 服务”。

    • 所述监视器指定类的所有子类部分显示定义了类的搜索范围的类。

    • “ 注释”过滤器部分显示用于匹配您感兴趣的类的特定实现的注释。

    • 单击“ 编辑范围”按钮以查看进程组中可用的进程。

    • 选择包含所需入口点的进程。

    • 要查看该类中存在的所有注释，请单击“ 继续”。

    1. 选择自定义服务类提供的一个或多个注释。

    2. 要扩大类匹配，请选择接口或超类作为匹配的基础。

    3. 要加载超类的继承，请单击“ 加载继承”按钮并展开注释列表。

    所有超类中的简介匹配

    此功能在所有超类和接口中查找指定的注释。这使得间接匹配成为可能，其中注释位于与接口匹配不直接相关的基类上。

    例如，查看以下类层次结构：

    code block public interface component { boolean executeService(); } @MyComponent public abstract class BaseServiceA { } public class MyServiceA extends BaseServiceA implements Component { }

    虽然您可以基于 Component 接口方法轻松定义自定义服务executeService，但您可能只需要那些具有MyComponent注释的实现。这样去做：

    1. 为其定义自定义服务MyServiceA。在这里，MyServiceA充当模板。

    2. 编辑范围以使用Component界面和MyComponent注释。

    按应用程序匹配注释

    许多框架仅依赖于注释。在这种情况下，您可能无法始终定义接口或基类。为了处理这种情况，Dynatrace 使您能够监视与特定包中的注释过滤器匹配的所有类。

    “ 编辑范围”页面为此明确目的提供了一个附加选项：使用带有选定注释的所有类和接口以及以...开头的名称。在选择注释之前， 此选项将显示为灰色。

    选择注释后，可以选择“ 使用所有类和接口...”选项并指定包匹配而不是接口。然后，Dynatrace 查找以给定字符串开头的任何类，该字符串在其自身或其任何超类和接口中具有所选注释。
