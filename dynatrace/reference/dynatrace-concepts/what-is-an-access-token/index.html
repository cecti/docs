    什么是访问令牌

    Dynatrace 依赖于惟一的字符串来标识监视环境。例如，这对于将 Dynatrace 监控环境与第三方消息服务、Dynatrace REST API、PaaS 环境、VoiceOps/ChatOps 和其他报告工具进行外部集成是必要的。

    对 Dynatrace 监控环境的所有外部访问都依赖于两个主要信息：环境 ID 和访问令牌。

    使用称为令牌的访问密钥可以实现 Dynatrace OneAgent 下载或访问 Dynatrace API 所需的身份验证。Dynatrace 定义了令牌的范围，以便出于安全原因限制对特定产品功能的访问。例如，PaaS 令牌允许下载 OneAgent for PaaS 并在多个主机上安装 OneAgent，但它不允许您访问其余的 Dynatrace API。

    在某些情况下，必须自动创建所需的令牌，以及必须手动生成令牌。例如，OneAgent for Windows 会自动附带令牌。相反，OneAgent for PaaS 要求您手动生成访问令牌。

    创建用户访问令牌

    生成和管理访问令牌

    1. 登录Dynatrace环境，然后从导航菜单中单击设置>集成。

    2. 选择 Dynatrace API，Platform as a Service 或 Dynatrace 模块以生成 Dynatrace API 的令牌，PaaS 的令牌或 DCRUM 或 Synthetic 的令牌。

    3. 单击“ 生成标记”按钮，然后在文本字段中键入有意义的标记名称。

    注意：不检查用户生成的令牌名称的唯一性。可以存在多个同名 API，PaaS 或模块令牌。请务必为您定义的每个标记提供有意义的名称。多个同名令牌可以具有不同的访问配置。有效的令牌命名可以帮助您有效地管理令牌，搜索令牌，并且如果不再需要它们，将来可能会将其删除。

    1. （可选）要创建 Dynatrace API 令牌，请根据需要选择或清除访问开关，以设置 API 令牌的访问范围。例如，要创建 API 身份验证令牌以访问用于用户会话查询的 Dynatrace 监控数据，请选择用户会话查询语言。

    2. 单击“ 生成”按钮。令牌出现在下面的 My Dynatrace 令牌列表中。
